Swim-Lane Activity Model

For example, during our last activity modeling session we identified two new use cases Perform Appraisal and File for Eminent Domain (see diagram). There's no point (at least at this time) to build out these use cases unless they contain information sources relevant to our architecture principles. Let's explore this diagram further.

The blue, rounded corner rectangles represent process for any given role in the swim-lane. The black arrows show the flow of control between activities (i.e., one activity ends and another begins). The green rectangles represent data (information object). You'll notice they have arrows pointing to input and output pins that control the object flow through the activity. These objects are either consumed or produced by the activity. You can't have an activity without both an input and an output object or it's not actually an activity. The position of an object (green rectangles) in any given swim-lane is not relevant to the swim-lane; rather, the pin flow is key to any shown object. Using the activity model, I was able to write a complete scope statement for this use case as follow:

Procure Easements Scope: When a representative from Real Estate Services receives a Request for Real Estate Services form, a Property Agent is assigned and the City Surveyor reviews the Client Department provided Field Notes for approval. Once approved, the Property Agent orders a Title Commitment and sends a Request for Appraisal to the City Appraiser to perform a Fair Market Value appraisal. Once complete, the Property Agent prepares an Offer Package to the Property Owner. Quality Assurance and Control reviews the Offer Package for adherence to current legislation and Senate Bill 18. When Quality Assurance and Control approves the Offer Package, it is sent to the Property Owner to sign the included Contract. If not signed within 45 days, another final offer of the approved Offer Package is sent to the Property Owner. If not signed within 15 days, proceedings to File for Eminent Domain obtains legal possession of the property. Finally, the Easement is signed, the Property Owner paid and the Easement is sent to the County Clerk for recording.

The swim-lane activity model is a useful instrument to communicate process to us architects and as you experienced, they are fairly easy to create. The activity model build allows us to capture your collective knowledge - i.e., the team comes to consensus on what the process looks like by contributing their understanding of the process. The data objects guide the architect to information flow understanding leading to the desired solution and fulfilling the desired architecture principles.
