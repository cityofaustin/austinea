Use Case Specification for Obtain Permits

Use Case Scope: Permit Applicant obtains single purpose trade permits or manages permits relevant to a Submittal Package/Project Information. The system provides Web interface application forms relevant to the permit requested, calculates required remittance and adds permit application to a Review Queue for Reviewer review and comment when required. Permit Applicants may also transfer Building Permits to prescribed contractors, cancel Trade Permits, and/or reactivate Expired Permits.

Table of Interface Descriptions

Category/InterfaceDescription
Issue Permit/
issueSinglePurposePermit		 The Permit Applicant selects the desired trade permit (or mobile home building permit) for the system to issue. This function is limited to only the alternate flow to request a permit and designated for selected trade contractors performing routine day-to-day activities requiring permits. When a mobile home is involved, this requires issuance of a building permit to associate relevant trade permits to the project. This type of building permit can be issued as a single purpose permit.
Issue Permit/
obtainPermit		 During the final review and approval of a Submittal Package, the review Notification provides a link (URL) with Project Information encoded in the link containing information that directs the Submittal Applicant to the correct location to obtain required permits. This message directs the Submittal Applicant to the proper URL where the system looks up relevant permit information.
Issue Permit/
lookUpPermitStatus		 The system looks up the Project Information encoded from the obtainPermit URL and presents the Submittal Applicant with options to submit for selected permits previously identified during the Perform Regulatory Review process. The Submittal Applicant owner (owner is defined as residing one calendar year with homestead exemption) is provided options to request a permit, transfer a building permit, cancel a trade permit, or reactivate an expired permit.
Issue Permit/
selectPermitOption		 The Permit Applicant selects from the permitting options available depending on the Permit Status of the project. The applicant may request a permit, transfer a building permit, cancel a trade permit, or reactivate an expired permit.
Issue Permit/
completePermitApplication		 Based on the Permit Applicant selected permit request, the system provides the appropriate permit application form for the Permit Applicant to compete and submit. Permit Applicant attribute values known by the system automatically populate relevant values on the form.
Issue Permit/
determineRemittance		 Based on the permit selected, the system automatically determines the Remittance required and presents the Remittance information to the Submittal Applicant with indication of selected form of payment (i.e., credit card, ACH, etc.). The Remittance amounts must be modifiable by identified city staff and not hard coded into the system.
Issue Permit/
payRemittance		 Using traditional online Electronic Financial Services approved by the city, the Submittal Applicant pays the required Remittance and receives Acknowledgment of payment.
Issue Permit/
addToReviewQueue		 The system provides a modifiable rules engine that places certain permit requests into a review queue for Reviewer to ensure completeness and relevance to the request. Common trade permits by established contractors may not require review.
Issue Permit/
reviewPermitApplication		 The Reviewer selects the Permit Application from the Review Queue and reviews for completeness and accuracy. If approved, the Permit Applicant is provided a URL encoded link to select to acknowledge inspection responsibility and permit issuance. If declined, the Notification provides indication of the deficiencies of the Permit Application and provides a URL encoded with information to redirect the Permit Applicant to the application to correct and resubmit.
Issue Permit/
acknowledgeInspectionResponsibility		 Once a permit is prepared by the system, the system indicates to the Permit Applicant responsibility for proper inspection and provides selected indication that this responsibility is accepted in the form of an Acknowledgment before a printable permit is issued by the system.
Issue Permit/
issuePermit		 When the system receives the proper Acknowledgment of inspection responsibility, the system prepares the selected permit in a printable form (i.e., PDF, etc.).
Permit Applicant/
printPermitAsRequired		 Once the Permit Applicant acknowledges inspection responsibility, the system provides a printable permit to display on construction site if required.
Issue Permit/
transferBuildingPermit		 There are circumstances when the permit owner (Permit Applicant) may wish to transfer a building permit to a prescribed contractor. The Permit Applicant enters relevant information into the system that clearly identifies the contractor receiving the transferred building permit and stores it into the system. Once transferred, the contractor plays the role of Permit Applicant to obtain future permits relevant to the Project Information associated with the case file building permit.
Issue Permit/
cancelTradePermit		 The Submittal Applicant (owner) can cancel trade permits already issued. Owner is defined as residing one calendar year with homestead exemption. If an inspection already occurred at the time of cancellation, there is no refund of Remittance; otherwise, a percentage of the Remittance is returned to Electronic Financial Service account. If a refund is issued, a new contractor must pay 100% of the required Remittance. If no refund issued, the contractor must pay 50% of the required Remittance to reissue the permit.
Issue Permit/
determineExpiredPermits		 The system determines the relevant expired permit(s) associated with the Submittal Package Project Information and presents to the Permit Applicant for selection to reactivate.
Issue Permit/
reactivateExpiredPermit		 The Permit Applicant uses the system to reactivate an expired permit. This requires a form to enter pertinent information (such as reason for reactivation) prior to reactivation.
