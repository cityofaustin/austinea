PDR Architecture Team,

It always amazes me how quickly the simplest process gets complicated. This is our challenge as we pursue successful information technology (IT) deployments. As an example, have a look at the traditional approach using a swim-lane model by selecting the diagram. The problem with this traditional "process" approach is the lack of an interface point of view. The interface is how our IT systems work; therefore, we need to explore functional needs from the interface perspective to clearly define and understand how we want to use our systems. By exploring the interface perspective, we're capturing system requirements the same way developers create systems or we can more effectively evaluate commercial-off-the-self (COTS) solutions because we clearly understand how we want to use our system. We can then evaluate how well the COTS solution implements our processes and determine if the COTS process constraints are acceptable to our business activities.

Use Case Specification
Perform Regulatory Review

1. Perform Intake

The Submittal Applicant inputs information and documents that represent a Submittal Package. For certain types of Submittal Packages, there's a fee required. Because a Submittal Applicant might intake more than one submittal during any given session, Remittance (fees) are stored in a Remittance Queue until the Submittal Applicant session ends.

3. Quality Review Submittal Package

The system provides a list of new or updated Submittal Packages for review by the Submittal Coordinator. The Submittal Coordinator reviews the Submittal Package and either rejects or accepts the submittal. If accepted, the system automatically adds the Submittal Package to the Group Review Queue. The type of Submittal Package determines the appropriate Group Review Queue entered and the order in which the groups must review the Submittal Package. If rejected, the system refunds a portion of the remittance and a portion of remittance is retained as a service fee. No matter the state of the Submittal Package, the Submittal Applicant is notified of the status of the submittal.

Type/InterfaceDescription
Perform Design Review/
reviewSubmittalPackage		 The system presents a Submittal Package Queue of new submittal packages and provides a means for the Submittal Coordinator to perform a quality check on the submitted items. Perform Design Review/
setupGroupReviewQueue		 After the Submittal Package is quality approved by the Submittal Coordinator, the system automatically identifies the Group Review Queue based on relevant application entries. The review cycle is adjustable by relevant city staff. For example, some groups may need to sign off (approve or comment) before the Submittal Package shows up in another Group Review Queue. The Group Review Queue is also updated when a Submittal Package is approved or requires additional review based on Reviewer lineage (requires one reviewer to review before another). The Group Review Queue is based on area of responsibility (i.e., constructability, code compliance, etc.). The Submittal Coordinator may return a Submittal Package to the Group Review Queue indicating selected groups during the review cycle to include Submittal Coordinator comments when necessary.
Submittal Applicant/
reviewNotification		 The system provides an automated email to the Submittal Applicant for accepted and rejected Submittal Packages (with rejection comment - i.e., reason for rejection and recommended guidance for future submissions). For accepted submissions, the system provides a summary of the submission process and estimation of anticipated review time. Throughout the Submittal Package review process, the system provides continuous automatic updates to the Submittal Application as it moves through the Group Review Queue.

4. Review Submittal Package

The system provides a means for a Reviewer to observe the Group Review Queue and select a Submittal Package for review. Once selected, the Group Review Queue is updated to remove the Submittal Package from the review queue to prevent simultaneous reviews. The Submittal Applicant is notified of the particular group reviewing the Submittal Package. Throughout the review cycle, a Submittal Coordinator may quality review the Reviewer Comment and provide feedback to the Reviewer by returning the Submittal Package back to the Group Review Queue to be addressed. Reviewer Comments made to the Reviewer are not provided as feedback to the Submittal Applicant. On completion of all areas of responsibility in the Group Review Queue, the Submittal Coordinator approves the Reviewer Comments for Submittal Applicant update or approves the Submittal Package for permitting.

provideDesignFeedback		 Routinely, the Reviewer in a particular review group (by order of responsibility area) checks the system Group Review Queue (listed in date/time order) for assigned reviews and submits Submittal Package Reviewer Comments. The system provides automated tools for the Reviewer to markup diagrams including comments, links to references and city codes, among other traditional markup indications to effectively communicate to the Submittal Applicant the required changes to relevant Submittal Package items. All markups indicate the particular Reviewer for identification by the Submittal Coordinator but not identified when submitted to the Submittal Applicant. If no comments, the Reviewer approves the Submittal Package and the Submittal Coordinator approves the Reviewer Comments or Submittal Package when no comments require updates to the Submittal Package. The Group Review Queue is per review group (depending on the type of review) and when under review (selected by a Reviewer), the system automatically takes the Submittal Package off the Group Review Queue to prevent multiple reviews by the same area of responsibility. For some reviews, depending on certain attributes identified on the application form, the system provides a priority order of review groups because some review groups may be required to review the Submittal Package and approve or comment before other groups are provided opportunity to review. The business rules for review group order must be modifiable by selected responsible city staff.
