Cancel Event
Use Case

Use Case Scope

The Event Representative can, at any time in the booking process Cancel their event. The Sales Manager may also need to cancel an event. The Sales Manager Accesses the system and the associated customer record, to annotate the reason for the cancellation. If the EVENT is set to PROSPECT, the Sales Manager can change the status to CANCELLED.

If it's in a TENTATIVE state, the Sales Manager schedules an Activity (trace) with the contract Administrator to release space, Cancel the event in system, and all related documents /calendar entries, etc are marked as CANCELLED. Notifications are sent internally to the all Staff (including accounting) already involved in the event planning. The Sales Manager chooses whether the system should automatically generate a "Cancellation" notice, which is sent to the Event Representative (sometimes the Sales Manager will manually construct their own personalized message.)

If it's in a DEFINITE state (contract executed), the Sales Manager schedules an Activity (trace) with contract Administrator to release space, Cancel the event in system, and all related documents /calendar entries, etc are marked as CANCELLED. Notifications are sent internally to the all Staff already involved in the event planning. Contract Administration will Notify Accounting whether deposits received should be returned. The Contract Administrator will ask the Sales Manager to check if a RESALE clause is in the contract and the financial implications. The Sales Manager chooses whether the system should automatically generate a "Cancellation" notice, which is sent to the Event Representative (sometimes the Sales Manager will manually construct their own personalized message.)

Cancel Event Interface Descriptions

Type/InterfaceDescription
Establish Event Booking/
cancelEvent		 The Sales Manager requests that the System cancels the Event. Sales Manager provides the System with event name, dates, reason for cancellation, lost to (if applicable). Possible reasons for cancellation (customizable list in the System):(Lost to another facility, Event cancelled, Event postponed due to budgetary reasons, Flights (airfare). Also the system provides a mechanism for recording to whom the event was lost: Lost to {customizable list in the System of other facilities, customizable by system users} San Antonio, Forth Worth etc.
Sales Manager/
cancelEvent		 The Event Representative notifies the Sales Manager to cancel event. Event name and date must be provided by Event Representative. Notification must be sent in writing (email) and must be from Customer instead of a third party.
Establish Event Booking/
requestReleaseOfSpace		 The Sales Manager requests the System notify the Contract Administrator to release the space reserved for a previously entered Event.
Establish Event Booking/
releaseSpace		 The Contract Administrator requests the System releases the space for the event. The Contract Administrator provides the System with the event name and dates. The System cancels the activities associated to the event.
Establish Event Booking/
scheduleActivitiesForInternalStaff		 They system generates numerous traces (activities/reminders). This is customizable/configurable and role-based. Traces (Activities/reminders) for Sales and Operations.
Establish Event Booking/
createCancellationNotice		 The Sales Manager requests that the System generates a cancellation confirmation notification to the Event Representative. The notification is created from a template which includes the event name, dates, and contract ID.
Sales Manager/
reviewContractTerms		 The Contract Administrator verifies that Sales Manager has reviewed the terms of the contract relative to a deposit.
Sales Manager/
discussPotentialFundReturnOwedKeptWithEventRepresentative		 The Sales Manager determines the potential return of deposited funds. Sales Manager confers with Finance. If resale occurs then trigger is required to review at date of event to return deposit.
Establish Event Booking/
createReminderForAccountingToReviewResale		 The Sales Managers requests the System create a reminder for the Accounts Receivable to review the resale clause for Event at date of Event.
Contract Administration/
engageAccountsReceivableForFinalContractMonies		 The Sales Manager requests the Contract Administrator engage Accounts Receivable to finalize actions required: return money, bill Event Representative for money owed, or keep money.
Establish Event Booking/
issueReturn		 The Contract Administrator uses the System to request issue of return to Event Representative. Contract ID and amount are included to process return.
Establish Event Booking/
requestApprovalOfReturnBySalesDirector		 The System sends a notification to the Sales Director requesting approval of return. Request includes Contract ID and amount of return.
Establish Event Booking/
approveReturn		 The Sales Director uses the System to approve the return amount for the specified Contract ID.
Accounts Receivable/
issueReturn		 The System requests that Accounts Receivable issues a return to Event Representative. The contract ID, customer, amount, and address are required to complete a return.
Establish Event Booking/
recordMoniesSent		 Accounts Receivable uses the System to record amount of monies returned to the Event Representative noting Contract ID.
Establish Event Booking/
requestBillForEventRepresentative		 The Contract Administrator uses the System to request a Bill for the Event Representative noting Contract ID, amount due, and reason for bill.
Establish Event Booking/
requestAccountsReceivableCreateBill		 The System sends a notification to Accounts Receivable with a request to create a bill containing Contract ID and amount due.
Establish Event Booking/
createBillForEventRepresentative		 Accounts Receivable uses the System to create a bill for the Event Representative using the Contract ID, amount due, and reason for bill.
Establish Event Booking/
deliverBillToEventRepresentative		 The System sends the Event Representative a bill containing Contract ID, event name, amount due, due date, and reason for bill. The System allows the Contracts Administrator and Sales Manager to evaluate the status of the bill (unpaid, paid), collection notices, and other actions taken on bill.
Establish Event Booking/
notifyAccountsReceivableOfMoniesRetained		 The Contract Adminstrator accesses the System and indicates the deposits for the cancelled event are retained.
Establish Event Booking/
notifyAccountsReceivableAboutMoniesRetained		 The System sends an internal notification to Accounts Receivable team indicating monies received prior to the Cancellation of Event should be retained.
Provide Functional Conference and Facility Space/
documentMoniesReceived		 The Accounts Receivable team accesses the system and annotates monies received due to Cancellation.
