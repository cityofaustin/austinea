Create Work Order
Use Case

Use Case Scope

A Work Order Requester accesses the system through a web portal. They Requester enters a request. The Materials Planner quality reviews the Work Order Request and creates a work order if one is needed. The system pre-populates the information from the work order request to the work order. If the Materials Planner needs more information, the Materials Planner provides comments on the request and the Requester is notified additional information is needed. If the Materials Planner determines the request is a duplicate or an inappropriate request, the Materials Planner provides comments on the request and the Requester is notified the order request has been declined. The system can automatically create a work order from a predefined preventative maintenence schedule. The Requestor can be a Caterer, Tenants, or Staff.

submitWorkOrderRequest		 The Work Order Requester access a web portal and enters the Work Order Request information. They system provides the ability to store and retrieve Work Order Request information for future edits and resubmission. If the Work Order Request is returned for additional information, the Work Order Request is once again editable until resubmission. The System pre-populates the name, phone number, supervisor name, area/building, title, etc on the Work Order Request of the Requester based on login account information.
