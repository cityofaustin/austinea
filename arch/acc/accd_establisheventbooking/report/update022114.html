Create Event Invoice
Use Case

Use Case Scope

At the end of the event the Event Coordinator inputs all the charges to be billed to the client. The Work Order Reviewer verifies and approves the work orders. Accounts Receivable retrieves charges input by Event coordinator,applies discounts (room night credits, Booking credits, Food and Beverage credits), factors in Deposits, adds Utility client invoice, verifies all information, and uses the System to create a final Invoice. A hard copy of the final invoice is mailed to the client, and a copy of the final invoice is made for the Event Coordinator. Once the invoice is received, the customer can contact the Event coordinator to discuss perceived issues with the Invoice. The Event Coordinator will make a judgment call and either change charges associated with the Work Order (triggering the creation of a new Invoice), or indicate to the customer the Invoice is as delivered.

Create Work Order Interface Descriptions

Type/InterfaceDescription
Event Coordinator/
enterClientCharges		 The Event Coordinator accesses the system and enters equipment charges, labor charges, security charges, parking charges (when applicable), discounts, facility rentals, damages, and other misc charges. And indicate the Customer is ready to be billed.
Workorder Reviewer/
approveWorkorderCharges		 Work order Reviewer is sent a notification (email/text) that work order items are ready for approval. The Workorder Reviewer examines the contract, and compares it with the charges entered for the Workorder. If errors are found, they contact the appropriate coordinator(s) and manipulate/change the Workorder until it is complete and ready for Invoicing. The Workorder Reviewer indicates the Workorder is ready for invoicing and a notification is sent to Accounts receivable to create the final invoice.
Manage Event Revenue/
requestCustomerInvoiceCreation		 The System notifies the Account Receivable Team all elements for a Work Order are entered and it is time to create a customer invoice.
Manage Event Revenue/
createInvoice		 Accounts receivable reviews the Workorder items, ensuring the client information (address, contact name, etc. are correct) and marks the terms of NET 30 for payment due. The invoice includes a preformed address label/Payment coupon to the Accounts Receivable. Accounts Receivable inputs the date the invoice is mailed. Creates a PDF version of the invoice, and dispatches a copy of the invoice to the Event Coordinator. If the client is okay with soft copy the invoice is emailed to the client contact, otherwise a hard copy is mailed.
Customer/
payInvoice		 The Customer receives the event invoice in the form desired (email or us postal service delivery), and evaluates the charges compared to the contract and determines if there are issues. If there are issues, they notify the event coordinator of the issues, possibly triggering a revised invoice creation. Note: Having the Customer be able to view Workorder items online via a web portal, is desirable. Not all customer issues result in invoice modification.
Event Coordinator/
adjustInvoiceCharges		 The Event coordinator receives notification from the customer about invoice specifics. The Event coordinator reviews the contract, the online Workorder and attempts to resolve the concern. Should changes be necessary, the Event coordinator makes adjustments to the Workorder and begins the "create Event Invoice" process again.
Manage Event Revenue/
createInvoiceChangeMemo		 The Event Coordinator determines adjustments (credits or debits) are necessary for Invoice. The Event Coordinator fills out the adjustment indicating Work Order Item indicating debit or credit amount, the justification for the change, the Contract ID for which this applies, and any supporting documentation.
