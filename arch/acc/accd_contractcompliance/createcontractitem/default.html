Create Contract Deliverable Use Case Model

Scope:

The Contract Item Creator uses the Contract Manager system to display a Contract by searching for the ContractVendorName or the ContractNumber and creates a ContractItem, entering ContractItemType (Deliverable or Term & Condition) and entering either ContractDeliverableInfo or ContractTermConditionInfo. If applicable, the Contract Item Creator next stores ContractDeliverableFrequencyInfo and clicks a button to generate one or more ContractDeliverableDates.

Provide Functional Conference and Facility Space/
storeContractDeliverableInfo

If the ContractItem is of ContractItemType 'Deliverable', the Contract Item Creator uses the system to store the following ContractDeliverableInfo: ContractID, SectionTitle (eg. Rights & Obligations. An editable value list), SectionTitleOrder, Section (eg: 1, 1.2, 1a, 2b), OrderNumber (to order the deliverables visually), DeliverableType (Approval, Notice, Report, Remedy, Agree, Renegotiate, etc from an editable value list), From (company from which the deliverable comes), To (ACCD), Title (a shorter version of the Description with character limitation 128), Description, Comments. From and To function interchangeably. Ex: Levy to ACCD or vice versa. Note: Have a screen where deliverable Title and Description are searchable.

Provide Functional Conference and Facility Space/
storeFrequencyInfo

The Contract Item Creator uses the system to store the ContractDeliverableFrequencyInfo: StartDate, EndDate, Frequency (once, bi-weekly, monthly, quarterly, biannually, annually), ApprovingReviewingEntity (an editable value list), ApprovingReviewingEntityEmailAddress (a lookup based upon the chosen entity). The ContractDeliverableFrequencyInfo is not required. A ContractDeliverable may have a single DueDate, multiple DueDates or no DueDate at all. The Contract Item Creator has the ability to change from one DueDate to multiple DueDates or vice versa, so long as ContractDeliverableDate VerificationStatus is Pending.
Provide Functional Conference and Facility Space/
createDeliverableFrequencyDate

When all ContractDeliverableFrequencyInfo has been entered, the Contract Item Creator clicks a button to generate one or more ContractDeliverableDates. Each Contract DeliverableDate (DateDue) also has an associated DateReceived, VerificationStatus (set by default to a value of 'Pending'), an ApprovingReviewingEntityName. The next chronological DateDue displays as the value of the NextDueDate in the ContractDeliverableInfo. The ApprovingReviewingEntity is the employee responsible for indicating that the ContractItem has been received. The ApprovingReviewingEntityEmail is used by the system to send an email notification. The Contract Item Creator has the ability to add an individual ContractDeliverableDate after a series of dates has been created. The Contract Item Creator may delete an individual ContractDeliverableDate, so long as the VerificationStatus is 'Pending'. The Contract Item Creator has the ability to click a button to delete all ContractDeliverableDates at once, but only those that have a VerificationStatus of 'Pending'. The Contract Item Creator has the ability to regenerate ContractDeliverableDates and these are added to the existing list of ContractDeliverableDates, if one already exists. The system does not automatically remove ContractDeliverableDates outside of the range of the newly created ContractDeliverableDates, neither does it remove duplicates. In such a case, the Contract Deliverable Create will manually/individually remove unneeded ContractDeliverableDates.
