Place Lien on Property
Use Case Model

Scope: The Austin Code Agent determines the need for placing a lien on the property after 30 calendar days from the creation of a bill (invoice). The Austin Code Program Specialist uses the Case Management System to create a LienDocument. The Case Management System updates the property to attach the LienDocument to the property. The Austin Code Program Specialist prints the LienDocument, checks to ensure property is not homesteaded (does nothing if homestead exemption is present), and submits the LienDocument to the Chain of Command (Austin Code Admin Manager) for approval. If the lien is approved, the Chain of Command (Austin Code Admin Manager) initials and submits to the next in Chain of Command (Austin Code Director). If the LienDocument is not approved, the Chain of Command (Austin Code Admin Manager) returns the LienDocument to the Austin Code Program Specialist with notes on required changes. The Austin Code Program Specialist recreates the LienDocument until it is approved. After the Chain of Command (Austin Code Director) signs and returns the LienDocument, the Austin Code Program Specialist completes the InterDepartmentPurchaseForm (which includes: list of liens, how many are filed at once, how many liens, filing fees, etc.). The Austin Code Program Specialist sends the InterDepartmentPurchaseForm to the Austin Code Admin Manager to sign off on the form. The Austin Code Program Specialist takes the InterDepartmentPurchaseForm, and all associated LienDocuments, to the Travis County Clerk. The Travis County Clerk prints a receipt for all FilingFees and stamps the form. The LienDocuments remain with the Travis County Clerk. The Austin Code Program Specialist submits the InterDepartmentPurchaseForm to Austin Code Finance to process payment to the Travis County Clerk. After the lien is created by Travis County Clerk, the lien is returned to the Austin Code Program Specialist-included is the TRVNumber. The Austin Code Program Specialist adds the TRVNumber to the CodeViolationCase indicating the lien was successfully placed on the property.

Place Lien on Property
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Case Management System/createLienDocument
Thirty (30) calendar days after the bill is issued, the Austin Code Program Specialist uses the Case Management System to generate a lien for a property. The Austin Code Program Specialist provides the Owner and property. The Case Management System generates a printed copy of the lien. The Austin Code Program Specialist reviews the lien and only updates the formatting. Data issues are managed through the Case Management System and the lien is regenerated.
002 Chain of Command/approveLienDocument
The Austin Code Program Specialist requests the Chain of Command (Austin Code Admin Manager) to review and approves the lien. The Austin Code Program Specialist delivers a hard copy of the lien. The Austin Code Admin Manager checks the Lien for accuracy and then initials the lien. Once approved, the Austin Code Admin Manager forwards the lien to the Austin Code Director for final approval and signature. The signed lien is returned to the Austin Code Program Specialist.
003 Austin Code Program Specialist/createInterDepartmentPurchaseForm
The Austin Code Program Specialist creates an InterDepartmentPurchaseForm in Word. The Austin Code Program Specialist checks the County Clerk Office for fee schedule.
004 Austin Code Admin Manager/approveInterDepartmentPurchaseForm
The Austin Code Program Specialist delivers the InterDepartmentPurchaseForm to the Austin Code Admin Manager (hard copy with liens attached). The Austin Code Admin Manager reviews the InterDepartmentPurchaseForm and signs if approved. If changes are required, the Austin Code Admin Manager returns the InterDepartmentPurchaseForm to the Austin Code Program Specialist for immediate update/modification.
005 Travis County Clerk/createLien
The Austin Code Program Specialist requests the Travis County Clerk to file a lien for the requested property. The Travis County Clerk FiledStamp (InstrumentNumber) the lien. The Austin Code Program Specialist also provides two copies of the InterDepartmentPurchaseForm. The Travis County Clerk keeps one copy, and stamps (DateStamp) the other copy for return to the Austin Code Program Specialist. The Travis County Clerk returns a receipt for the filed liens. A TRVNumber is returned.
006 Austin Code Finance/makePayment
The Austin Code Program Specialist requests Austin Code Finance to process payment for the lien. The Austin Code Program Specialist includes the receipt for the lien along with the InterDepartmentPurchaseRequestForm. The documents are sent through interoffice mail.
007 Case Management System/updateCodeViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase.
