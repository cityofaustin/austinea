Release Lien On Property
Use Case Model

Scope: The City Legal Department contacts the Austin Code Program Specialist to request a release of a Lien on a Property, if the Property was foreclosed. The City Legal Department provides a WarrantyDeed as proof of the foreclosure. If the Property is not foreclosed, a Responsible Party contacts the Austin Code Program Specialist and requests a PayoffStatement for the Lien. The Austin Code Program Specialist provides the PayoffStatement and the Responsible Party submits payment. The Process Payment use case is started once the payment arrives. After payment has been processed the Austin Code Program Specialist updates the CodeViolationCase in the Case Management System with notes on the payoff. The Austin Code Program Specialist also updates the Lien in the Case Management System, attaching the PayoffStatement and WarrantyDeed as appropriate. The Austin Code Program Specialist creates a ReleaseOfLien document and submits to the Chain of Command for approval. The Austin Code Program Specialist submits the ReleaseOfLien to the Chain of Command, and the Chain of Command signs the document. The Austin Code Program Specialist then creates an InterDepartmentPurchaseForm, and submits the InterDepartmentPurchaseForm to the Austin Code Admin Manager for approval. The Austin Code Admin Manager approves the InterDepartmentPurchaseForm and returns to the Austin Code Program Specialist. The Austin Code Program Specialist requests the Travis County Clerk release the Lien. After the Lien is released the Austin Code Program Specialist requests Austin Code Finance may Payment to the Travis County Clerk. The Austin Code Program Specialist updates the CodeViolationCase to indicate the Lien was released.

Release Lien On Property
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Austin Code Program Specialist/releaseLienPerForeclosure
The City Legal Department notifies the Austin Code Program Specialist that a Lien must be vacated due to a foreclosure. Notification is typically provided through email and contains the WarrantyDeed identifying the Property.
002 Austin Code Program Specialist/issueLienPayoffStatement
The Responsible Party contacts the Austin Code Program Specialist and requests a PayoffStatement for the Lien related to a Property.
003 Responsible Party/submitPayment
The Responsible Party submits Payment for the Property Lien.
004 Case Management System/Update Code Violation Case
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase. The Austin Code Officer updates the CodeViolationCase with information concerning Violation extensions, written request, reason for approval/denial, etc. If there is a payment, then the CodeViolationCase is updated by the user to include the payment.
005 Case Management System/updateLienDocument
The user accesses the system and updates the Lien. If a PayoffStatement was provided the user uploads the PayoffStatement. If a WarrantyDeed was provided to vacate the Lien, then the WarrantyDeed is attached.
006 Austin Code Program Specialist/createReleaseOfLienDocument
The Austin Code Program Specialist creates a ReleaseOfLien document. The ReleaseOfLien identifies the Lien and the Property.
007 Chain of Command/ApproveReleaseOfLien
The Austin Code Program Specialist submits the ReleaseOfLien document to the Chain of Command for approval. The Chain of Command reviews the ReleaseOfLien, signs, and returns to the Austin Code Program Specialist.
008 Austin Code Program Specialist/createInterDepartmentPurchaseForm
The Austin Code Program Specialist creates an InterDepartmentPurchaseForm in Word. The Austin Code Program Specialist checks the County Clerk Office for fee schedule.
009 Austin Code Admin Manager/approveInterDepartmentPurchaseForm
The Austin Code Program Specialist delivers the InterDepartmentPurchaseForm to the Austin Code Admin Manager (hard copy with liens attached). The Austin Code Admin Manager reviews the InterDepartmentPurchaseForm and signs if approved. If changes are required, the Austin Code Admin Manager returns the InterDepartmentPurchaseForm to the Austin Code Program Specialist for immediate update/modification.
010 Travic County Clerk/releaseLien
The Austin Code ProgramSpecialist requests theTravis County Clerk release the Lien for the Property. The Austin Code Program Specialist provides the TRVNumber, Lien, ReleaseOfLien document, and if appropriate, the WarrantyDeed. The Travis County Clerk releases the Lien and returns a LienReceipt identifying the Lien was released.
011 Austin Code Finance/makePayment
The Austin Code Program Specialist requests Austin Code Finance to process payment for the lien. The Austin Code Program Specialist includes the receipt for the lien along with the InterDepartmentPurchaseRequestForm. The documents are sent through interoffice mail.
