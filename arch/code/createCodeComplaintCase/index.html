Create Code Complaint Case
Use Case Model

Scope: Violations are identified either by Complainants or Austin Code Officers. If a Complainant identifies the Violation, the Complainant contacts Austin 311 to report the Violation. Austin 311 uses the CSR system to create an Austin Code ServiceRequest (ServiceRequestAustinCode). If the ServiceRequest involves a short-term rental, the CSR system sends a Notification to the Austin Code Officer. ARR Customer Service queries the CSR system for new Austin Code ServiceRequests, and then creates a CodeComplaintCase for each ServiceRequestAustinCode in the queue within the Case Management System. Alternatively, Violations are identified by Austin Code Officers while out in the field. Once the Violation is identified, the Austin Code Officer notifies ARR Customer Service to create a CodeComplaintCase. ARR Customer Service uses the CSR system to create a ServiceRequestAustinCode for the Violation (primarily for tracking and reporting purposes), and then creates the CodeComplaintCase in the Case Management System. ARR Customer Service assigns an Austin Code Officer (AssignedCodeOfficer) and Austin Code Program Specialist (AssignedProgramSpecialist) to the CodeComplaintCase. ARR Customer Service then updates the ServiceRequestAustinCode with the CodeComplaintCase information and closes the ServiceRequest. If the CodeComplaintCase needs to be reassigned, the Chain of Command reasisgns the CodeComplaintCase to the appropriate Austin Code Officer. This is primarily a district and/or zoning alignment.

Create Code Complaint Case
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Austin 311/reportCodeViolation
The Complainant contacts Austin 311 to report a Violation and request a ServiceRequestAustinCode be created. Austin 311 creates the requested ServiceRequest and provides the Complainant with a ServiceRequestNumber to track the progress of their request until a CodeComplaintCase is created.
002 CSR/createAustinCodeServiceRequest
The user uses the CSR system to create a ServiceRequest for Austin Code (ServiceRequestAustinCode). If a Complainant reported the Issue (Violation), Austin 311 returns a ServiceRequestNumber to the requestor.
003 CSR/sendNotification
The CSR system sends a notification to the Austin Code Officer with description of issue, issue type, complaint SR, SR number, date entered, reported location, comments, nature of concern, time happening, date happening, type of short-term rental, and geo information).
004 CSR/queryQueueOfServiceRequests
ARR Customer Service uses the CSR system to query (CSRQuery) for new ServiceRequests for Austin Code (ServiceRequestAustinCode). The system returns new ServiceRequestAustinCode records.
005 Case Management System/createCodeComplaintCase
ARR Customer Service uses the system to create CodeComplaintCases for each ServiceRequestAustinCode in the queue.
006 ARR Customer Service/createCodeComplaintCase
TThe Austin Code Officer requests ARR Customer Service, typically through email, to create a CodeComplaintCase based on their observation of a Violation from the field.
007 Case Management System/assignInspectorToCase
The Austin Code Officer and the Austin Code Program Specialist are assigned to the case simultaneously. Both are assigned based on location/district.
008 CSR/updateServiceRequest
The user updates the ServiceRequest with notes concerning their resolution of the ServiceRequest. For Austin Code ServiceRequests (ServiceRequestAustinCode), the ServiceRequest is closed once the CodeComplaintCase is created and assigned to an Austin Code Officer (AssignedCodeOfficeR).
009 CSR/closeServiceRequest
The ServiceRequest is closed by a user or the system. Closed ServiceRequests may not be edited by users or reopened per the user's access permissions.
