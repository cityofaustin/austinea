Serve Warrant
Use Case Model

Scope: The Austin Code Officer requests a Notary to notarize an Affidavit to complete a SearchWarrant. After the Affidavit has been notarized, the Austin Code Officer submits the SearchWarrrant, with Affidavit attached, to a Court Judge (either Municipal Court or District Court Judge) for approval. The Court Judge reviews the SearchWarrant and determines whether to sign the SearchWarrant and make it active, or not sign it. The Court Judge returns the SearchWarrant with their decision. If the SearchWarrant is signed by the Court Judge, the Austin Code Officer requests an APD District Representative serve the SearchWarrant. The APD District Representative serves the SearchWarrant within three calendar days. If the SearchWarrant is not served within three calendar days, it is no longer active and must be resubmitted to the Court Judge for approval. If the SearchWarrant is served, the Austin Code Officer performs an Inspection, and creates an InventoryList during the Inspection. The InventoryList and SearchWarrant are returned to the Court. Afterwards, the Austin Code Officer updates the CodeViolationCase in the Case Management System with the SearchWarrant, InventoryList, Inspection results, and their notes.

Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Notary/notarizeAffidavit
The Austin Code Officer requests a Notary within the Austin Code Department to notarize the Affidavit. The Notary notarizes the Affidavit and returns to the Austin Code Officer.
002 Court Judge/approveSearchWarrant
The Austin Code Officer requests the Court Judge, either District Court or Municipal Court, to approve the SearchWarrant. If the SearchWarrant is approved, the Court Judge signs the SearchWarrant and the SearchWarrant becomes active. If the Court Judge does not approve the SearchWarrant, the SearchWarrant is left unsigned. The Court Judge returns the SearchWarrant to the Austin Code Officer.
003 APD District Representative/serveSearchWarrant
The Austin Code Officer requests the APD District Representative to assist serving the SearchWarrant. If the APD District Representative is unavailable, an alternative APD Officer is used to serve the SearchWarrant. The APD District Representative serves the SearchWarrant within three calendar days. If the SearchWarrant isn't served within three calendar days, the SearchWarrant is not considered active and must be resigned by a Court Judge. The SearchWarrant is signed by the APD Officer witnessing the SearchWarrant's service.
004 Austin Code Officer/performInspection
If the violation concerns hazardous conditions, the Austin Code Officer applies Personal Protective Equipment (PPE). The Austin Code Officer inspects the Property. The Austin Code Officer inspects from the exterior (initially from the right of way) to the interior (if the interior is accessible and permission has been granted by legally interested party or someone in Care/Control of Property). The Austin Code Officer takes photographs and measurements as needed (hot water temperature, air quality, interior temperature, light meter, ground-fault circuit interrupter, gas meter, carbon monoxide, smoke detector, fences, grass height, driveway and walkways, checks impervious cover, etc.). As needed, the Austin Code Officer performs research to identify the Property Owner, offense, ViolationType, OwnerInformation, and relevant PropertyInformation (GIS system, ARR Site, annexation information, AMANDA permit history, site plan, TCAD/WCAD for OwnerHistory and DeedHistory, CC&B, ZoningHistory, CodeOfOrdinances, etc.). The Austin Code Officer interviews relevant parties. For Waste Hauler Inspection: The Austin Code Officer makes contact with driver, visually inspects vehicle for decals on door, asks driver about awareness requirement for waste hauler license, ensures driver is operating within City of Austin (outside of city is exempt), double checks the Waste Hauler license, checks for safety-equipment violations, driver's license of driver, and takes photos of truck (including both doors and front license plate). If a short-term rental complaint, the Austin Code Officer responds more quickly than other complaints. On Fridays and Saturdays, the Austin Code Officer performs inspection during day, and from 6:00 p.m. - 2:00 a.m. On Sundays, inspections occur from 8:00 a.m. - 12:00 p.m.
005 Austin Code Officer/returnInventoryListandSearchWarrantToCourt
The Austin Code Officer returns the InventoryList and SearchWarrant to the appropriate Court (Municipal Court or District Court).
006 Case Management System/updateCodeViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase. The Austin Code Officer updates the CodeViolationCase with information concerning Violation extensions, written request, reason for approval/denial, etc. If there is a payment, then the CodeViolationCase is updated by the user to include the payment.
