Order Property Abatement
Use Case Model

Scope:

A CodeViolationCase requires property abatement. If the Property is private and occupied the Obtain Municipal Court Search and Seizure Warrant use case starts. The Austin Code Officer obtains city legal approval, and then serves the SearchWarrant to identify the abatement required per the Create Abatement List use case. If the violation is in the right of way, or private and non-occupied a SearchWarrant isn't necessary and the Create Abatement List use case starts.

The Austin Code Officer creates a WorkRequiredList for a Contractor, and after approval the WorkRequiredList is assembled into a CutListPacket for the Contractor. If the abatement is completed by the Community Court the Austin Code Officer notifies Community Court of abatement cases. After the Property Abator, either Contractor or Community Court, has been notified the Abate Property with Contractor use case begins.

If the Property is a transient camp, or an occupied, the Property Abator abates the Property while the Austin Code Officer, APD, and other external entities are on-site to supervise. If the abatement doesn't match the requested work an abatement adjustment request is made.

If the Property isn't a transient camp, or occupied, the Property Abator abates the Property alone, and an Austin Code Officer inspects the Property afterwards.

In both instances the Property Abator (Contractor) creates an Invoice and sends to Austin Code. If no Invoice is received no further work is required for the Property abatement. If an Invoice is received the Pay Contractor and Bill Owner use case starts.

Austin Code verifies the Invoice, approves the amount through the Chain of Command, and pays the Contractor. If the Property abatement was for private Property a BillNotice is created in the Case Management System and the appropriate Liens/Receivables are created, and issued to the Owner.
