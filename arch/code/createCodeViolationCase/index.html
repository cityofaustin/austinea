Create Code Violation Case
Use Case Model

Scope: The Austin Code Officer looks up and verifies case information in the system. The Austin Code Officer then creates a CodeViolationCase in the system. The system updates the CodeComplaintCase to link it to the new CodeViolationCase. Several options are then available to the Austin Code Officer based on the disposition and requirements of the CodeViolationCase:

  • If emergency response is required, the Engage Emergency Response use case begins.
  • If the Violation involves zoning or land use, the Inspect Zoning Land Use Violation use case begins.
  • If the Violation involves a dangerous or substandard Violation, the Inspect Structure Code Violation use case begins.
  • If new Violations are identified at the same Location, the Austin Code Officer updates the CodeViolationCase with the new Violations.
  • If new Violations are identified at different Locations (typically a neighbor, etc.), the Create Code Complaint Case use case is used to create CodeComplaintCases for each Violation at the new Locations.
  • If the Property has a history of Violations, the Inspect Repeat Offender Violation use case begins.
  • If a Warning is warranted, the Austin Code Officer issues a Warning to the Violator.
  • If a Citation is warranted, the Austin Code Officer begins the Issue Violation Citation use case.
  • If a NoticeOfViolation hasn't been created previously for the Violation and is required, the Send Notice of Violation use case begins.
  • If Owner hardship is present, the Provide Community Outreach Assistance use case begins.
The Austin Code Officer updates the CodeViolationCase in the system with notes of their actions. Finally, if Complainant information is available and contact is warranted, the Austin Code Officer contacts the Complainant with their findings.

Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Case Management System/lookupAndVerifyCaseInformationolation
The Austin Code Officer requests the Case Management System to verify that the case information is accurate (i.e., Owner, Tenant, billing, etc.). CC&B is used to lookup Tenant information and verify property information.
002 Case Management System/createCodeViolationCase
The user creates a CodeViolationCase. If citation is created, it indicates the ticket number, person who wrote it, date of birth, and person cited (CT Folder in Amanda). No NoticeOfViolation is required if cited immediately without a CodeViolationCase. If Contractor cited, then NoticeOfViolation is sent to the Owner. Continuation of CodeComplaintCase, adding:
  • Deficiencies/Violations (actual code violated)
  • Remedy (time to correct, work required)
  • Photographs
  • Notes
  • Postings (DoorHangers, Placards, StopWorkOrders)
  • Residential/Commercial Designation
  • Type of Structure
  • Type of Violation (substandard, dangerous, work without permit, land use, etc.)
  • Occupied or Vacant
003 Case Management System/updateCodeComplaintCase
The user updates the CodeComplaintCase with information on the Complaint. If there's additional information provided by Complainant, it's added to CodeComplaintCase. The Austin Code Officer adds Inspection results to CodeComplaintCase including Measurements and Photographs. If a DoorHanger was placed on the Property, the Austin Code Officer adds notes for the DoorHanger. If other government agencies are contacted, the Austin Code Officer adds notes about agency contacted and services requested.
004 Case Management System/updateCodeViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase. The Austin Code Officer updates the CodeViolationCase with information concerning Violation extensions, written request, reason for approval/denial, etc.
005 Austin Code Officer/issueWarning
The Austin Code Officer issues a Warning to the Violator. Note: Warnings are non-legal notices. A NoticeOfViolation is still required after a Warning is issued.
006 Austin Code Officer/contactComplainantWithFindings
The Austin Code Officer contacts the Complainant (method of contact varies based on contact information available) to notify of whether a Violation was found or not. The Austin Code Officer provides an overview of the Code Process and Expected Actions, along with the Expected Resolution Time.
