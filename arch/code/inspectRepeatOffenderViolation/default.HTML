Inspect Repeat Offender Violation
Use Case Model

Scope: An Austin Code Officer identifies a potential Repeat Offender. The Austin Code Officer determines if the Property and Responsible Party qualify as a Repeat Offender; the Austin Code Officer reviews the Property case history, Citations, and NoticeOfViolations to see if Repeat Offender criteria has been met (two Citations in two years, or five NoticeOfViolations in two years, or two Violations not met in a timely manner in two years). The Austin Code Officer creates a RepeatOffenderPacket and sends the RepeatOffenderPacket to the Chain of Command for review. If the Chain of Command disagrees with the Repeat Offender identification, or requires changes, the use case ends. If the Chain of Command agrees with the Repeat Offender identification, the Chain of Command schedules a Repeat Offender evaluation with the ROP Review Committee and requests the ROP Review Committee evaluate the Repeat Offender. If the ROP Review Committee rejects the Repeat Offender identification, the use case ends. If the ROP Review Committee accepts the Repeat Offender identification, the Chain of Command requests the Austin Code Program Specialist update the RepeatOffenderList. The Austin Code Program Specialist then creates a RepeatOffenderCase in the Case Management System. The Austin Code Program Specialist sends notice to the Responsible Party to register as a Repeat Offender and updates the RepeatOffenderCase with the sent notice. If the Responsible Party appeals the Repeat Offender identification, they send an appeal Notification (letter) to Austin Code. The Chain of Command reviews the appeal. If the appeal is denied, then the Director reviews the appeal as a final check. The Austin Code Program Specialist sends the appeal decision to the Responsible Party (approved or denied). If the appeal was approved, the Austin Code Program Specialist updates the RepeatOffenderCase, closes the RepeatOffenderCase, and the use case ends. Once notified, the Responsible Party has 14 days to register. If the Responsible Party doesn't register within 14 days, the Austin Code Officer issues a Warning. The Austin Code Officer issues a Citation per the Issue Violation Citation use case, if the Responsible Party still doesn't register. The Responsible Party post Signs at the Property and submits the RepeatOffenderApplication to Austin Code, including payment. Austin Code Finance receives the payment and application, begins the Process Payment use case, and forwards the RepeatOffenderApplication to the Austin Code Program Specialist. The Austin Code Program Specialist records the Repeat Offender's registration and updates the RepeatOffenderCase. The Austin Code Program Specialist then sends the RepeatOffenderLicense to the Repeat Offender. After the Repeat Offender has registered, the Chain of Command requests the Public Information Office Admin update the RepeatOffenderWebsite. The Public Information Office Admin in turn requests the GIS Analyst to update the RepeatOffenderMap for use on the RepeatOffenderWebsite. The Austin Code Officer performs follow-up Inspections on the Repeat Offender, per Ordinance. The Austin Code Officer gathers information to include in the RepeatOffenderSummaryReport, and updates the RepeatOffenderCase in the Case Management System. If any Violations are identified during the Inspection, the Austin Code Officer starts the Create Code Violation use case. The Austin Code Officer then requests the Austin Code Program Specialist send the RepeatOffenderSummaryReport to the Owner and Responsible Party. If the Repeat Offender satisfies the requirements for the Repeat Offender program, the Austin Code Program Specialist closes the RepeatOffenderCase.

Inspect Repeat Offender Violation
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Austin Code Officer/evaluateRepeatOffender
The Austin Code Officer performs research on the Property (such as case history and NoticeOfViolation) to determine if someone is a Repeat Offender-criteria is two Citations in two years, or five NoticeOfViolations in two years, or two violations not met in a timely manner in two years.
002 Austin Code Officer/createRepeatOffenderPacket
The Austin Code Officer creates a Repeat Offender Packet, which includes checklist, TCAD information, all NoticeOfViolation, the reason for repeat offender (the 2/5/2 criteria), photographs, and case history.
003 Chain of Command/evaluateRepeatOffender
The Austin Code Officer sends the RepeatOffenderPacket to the Chain of Command for approval. The Chain of Command reviews the packet for completeness and either approves or disagrees (primarily to include additional information) the Repeat Offender identification. The Chain of Command returns the packet to the Austin Code Officer, and if approved, schedules a Repeat Offender consideration with the ROP Review Committee. If the Responsible Party appeals the Repeat Offender identification, the Chain of Command reviews the appeal. If the appeal is denied, the appeal is escalated to the Director for final review.
004 ROP Review Committee/evaluateRepeatOffenderCase
The ROP Review Committee reviews the 2/5/2 criteria as part of the RepeatOffenderPacket and makes a determination. The ROP Review Committee either accepts or rejects the Repeat Offender identification. The ROP Review Committee returns the RepeatOffenderPacket to the Chain of Command.
005 Austin Code Program Specialist/updateRepeatOffenderList
The Austin Code Program Specialist updates the RepeatOffenderList in Excel with the name of Property, address, Owner name, Property Manager name, LicenseNumber, and registration (complete or incomplete).
006 Case Management System/createROPCase
The Austin Code Program Specialist creates a ROPCase in the system, and provides Property, Responsible Party, reason for repeat offender, and the required registration letter is saved as an attachment.
007 Austin Code Program Specialist/sendNoticeToRegisterToResponsibleParty
The Austin Code Program Specialist sends notification via certified mail to the Responsible Party to register. The Responsible Party is instructed to display signs per Ordinance (i.e., signs must be at least 12"x12" with white and black contrasting and a contact for emergencies).
008 Case Management System/updateROPCase
The user updates the RepeatOffenderCase in the system with information on notifications sent to Responsible Party, appeal information, the application and fees received, inspection details, signed summary report, etc.
009 Responsible Party/sendAppealToChainOfCommand
The Responsible Party appeals the Repeat Offender identification by letter to the Chain of Command.
010 Austin Code Program Specialist/sendAppealDecisionToResponsibleParty
The Chain of Command requests the Austin Code Program Specialist sends the notification to the Responsible Party of the Chain of Command's decision of either approval of the appeal or denial.
011 Case Management System/closeROPCase
The system closes the RepeatOffenderCase.
012 Austin Code Officer/issueWarning
The Austin Code Officer issues a warning to the Violator. Note: Warnings are non-legal notices. A NoticeOfViolation is still required after a warning is issued.
013 Responsible Party/postSignsOnProperty
The Responsible Party posts signs at the Property (emergency number for point of contact, appropriate size, number of signs displayed based on percentage size of property, located in common areas and manager's office). Sign dimensions and criteria are based on the Ordinance.
014 Austin Code Finance/processRepeatOffenderApplication
The Responsible Party fills out a RepeatOffenderApplication (Owner, Owner Name, address, phone number, and Property Manager/Regional Manager/Owner information) and mails it to Austin Code Finance. RepeatOffenderApplication has a $100 registration fee.
015 Austin Code Program Specialist/recordRepeatOffenderRegistration
The Austin Code Program Specialist records the Repeat Offender registration on the RepeatOffenderList.
016 Austin Code Program Specialist/sendRepeatOffenderInformationToResponsibleParty
The Austin Code Program Specialist mails the RepeatOffenderLicense to the Responsible Party via certified and regular mail, and updates the RepeatOffenderList.
017 Public Information Office Admin/updateRepeatOffenderWebsite
The Chain of Command requests the Public Information Office Admin to update the RepeatOffenderWebsite.
018 GIS Analyst/updateRepeatOffenderMap
The Public Information Office Admin requests the GIS Analyst to update the RepeatOffendMap.
019 Austin Code Officer/performRepeatOffenderInspection
The Austin Code Officer inspects all exterior buildings, all vacant units, and the percentage of occupied units (usually around 5-10% of any violations). The Austin Code Officer creates a RepeatOffenderSummaryReport of the inspection for distribution to the Owner and Responsible Party. If a Repeat Offender doesn't complete their two-year probation, then an extra five years is added to the Repeat Offender.
020 Austin Code Program Specialist/sendRepeatOffenderSummaryReportToOwner
The Austin Code Officer requests the Austin Code Program Specialist to send, via certified and regular mail, the SummaryReport to the Owner/Responsible Party to sign.
