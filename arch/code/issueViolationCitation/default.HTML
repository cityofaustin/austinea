Issue Violation Citation
Use Case Model

Scope: The Austin Code Officer writes a citation, which requires a good form of ID. The Austin Code Officer provides a copy of the citation to the violator. If Austin Policy Department (APD) assistance is required to complete the citation, the Austin Code Officer requests an APD Officer's assistance. The Austin Code Officer takes photographs of what the violation citation was created for-one citation is generated per violation. The Austin Code Officer updates the Case Management System with citation information. The citation is scanned and uploaded into the Case Management System (sometimes by the Austin Code Officer, other times by Admin). Citations are attached to the relevant CodeViolationCase; if no case exists when citation is created, the Austin Code Officer creates a CodeViolationCase before uploading the citation. After the citation is created in the Case Management System, refer to the Initiate Court Action use case.

Issue Violation Citation
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Austin Code Officer/createCitation
The Austin Code Officer creates a citation. The Austin Code Officer completes the citation and hands the citation to the Offender. The Offender signs the citation (or may refuse to sign it), and the Austin Code Officer explains the citation. If the Offender refuses to sign (doesNotSign), the Austin Code Officer later indicates the refusal in the Case Management System. If the Offender refuses to cooperate or identify themselves, APD is engaged to assist with the identification.
002 APD Officer/assistWithCitation
The Austin Code Officer requests an APD Officer to provide assistance to complete a citation.
003 Case Management System/createCodeViolationCase
The user creates a CodeViolationCase. If citation is created, the ticket number, person wrote, date of birth, person cited (CT Folder in Amanda). No NoticeOfViolation required if cited immediately without a CodeViolationCase. If Contractor cited, then NoticeOfViolation is sent to the Owner. Continuation of CodeComplaintCase, adding:
  • Deficiencies/Violations (actual code violated)
  • Remedy (time to correct, work required)
  • Photographs
  • Notes
  • Postings (DoorHangers, Placards, StopWorkOrders)
  • Residential/Commercial Designation
  • Type of Structure
  • Type of Violation (substandard, dangerous, work without permit, land use, etc.)
  • Occupied or Vacant
004 Case Management System/createCitation
The Austin Code Officer requests the Case Management System to create a citation, which includes, ticket number, name of offender, date of birth, violation code, driver's license number and court appearance (30 calendar days from citation date-must be on a business day).
005 Case Management System/updateCodeViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase.
