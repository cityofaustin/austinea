Create Receivable
Use Case Model

Scope: Austin Code Finance receives an invoice indicating a party owes money to the City of Austin. Austin Code Finance creates a receivable in AIMS. Austin Code Finance assembles a ReceivablePacket and sends to the Chain of Command for approval. The Chain of Command reviews the ReceivablePacket, and if approved, approves the receivable in AIMS.

Create Receivable
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 AIMS/createReceivable
Austin Code Finance receives an invoice and then creates a receivable in the system with document description, record date, extended description (case management invoice number), internal reference number, vendor information, vendor legal name, vendor address, due date, billing profile, collection cycle, invoice instruction, billing type, AR dept (1600), AR unit (all), accounting line, event type, line type, line description, amount (matches invoice), fund, department, unit, and revenue code (starts with a 4). A receivable can have multiple accounting lines (event type, line type, line description, amount [matches invoice], fund, department, unit, and revenue code [starts with a 4]). The receivable is printed and attached to the invoice (ReceivablePacket). Once submitted, it goes to the Chain of Command for approval in AIMS-similar to the CashReceipt process.
002 Austin Code Finance/assembleReceivablePacket
Austin Code Finance assembles a ReceivablePacket that contains the printed receivable and invoice.
003 Chain of Command/approveReceivable
Austin Code Finance requests the Chain of Command to approve the ReceivablePacket. The Chain of Command reviews the ReceivablePacket and approves the Receivable in AIMS. Once the approval is complete, the Chain of Command returns the ReceivablePacket to Austin Code Finance.
004 AIMS/approveReceivable
The user approves the receivable in the system.
