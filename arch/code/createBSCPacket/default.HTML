Create Building Standards Commission (BSC) Packet
Use Case Model

Scope: An Austin Code Officer requests a Research Analyst creates maps of a Property for a Building Standards Commission case. The Austin Code Officer assembles a BSCPacket. The Austin Code Officer also creates a LegalCase in the Case Management System. The BSCPacket requires multiple levels of approval from the Chain of Command. The Austin Code Officer submits the BSCPacket to the Chain of Command (the Austin Code Officer’s direct manager/supervisor) for review. The Chain of Command reviews the BSCPacket and returns to the Austin Code Officer. If any changes were identified during the Chain of Command Review the Chain of Command requests the Austin Code Officer modify the BSCPacket. The Austin Code Officer updates the CodeViolationCase in the Case Management System after the review. The Austin Code Officer then submits the BSCPacket to the Chain of Command (Assistant Division Manager) for review. The Chain of Command reviews the BSCPacket. If any changes were identified during the Chain of Command Review the Chain of Command requests the Austin Code Officer modify the BSCPacket. The Chain of Command requests the Austin Code Legal Officer review the BSCPacket (typically completed within 72 hours of request). The Austin Code Legal Officer completes their review, and if changes are necessary, requests the Austin Code Officer modify the BSCPacket. The Austin Code Legal Officer then coordinates a BSC case review. The Austin Code Officer, Investigator, Assistant Division Manager, and Austin Code Legal Coordinator meet to discuss the BSCPacket contents in relation to the upcoming BSCCase.

Create Building Standards Commission (BSC) Packet
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Research Analyst/createBSCMapsForProperty
The Austin Code Officer contacts the Research Analyst and requests the creation of structure, location and road maps related to the Property for the Building Standard Commission case.
002 Austin Code Officer/createBSCPacket
The Austin Code Officer creates a BSCPacket. The BSCPacket contains a BSCPacketChecklist
003 Case Management System/createLegalCase
The user accesses the Case Management System and creates a LegalCase. The user identifies the case type (Admin Hearing, Building Standards Commission, Municipal Court, or District Court).
004 Chain of Command/reviewBSCPacket
The Chain of Command reviews the BSCPacket for completeness and if correct signs the BSCPacketChecklist. If changes are required the Chain of Command identifies gaps and returns the packet to the Austin Code Officer.
005 AustinCodeOfficer/modifyBSCPacket
The user sends an email to the BSC coordinator, the Assistant Division Manager, and manager indicating the missing required documentation (or any information the Austin code officer supplied.) The Austin Code Officer modifies the BSCPacket as requested. The updated BSCPacket is returned to the original requestor.
006 Case Management System/updateViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase. The Austin Code Officer updates the CodeViolationCase with information concerning Violation extensions, written request, reason for approval/denial, etc. If there is a payment, then the CodeViolationCase is updated by the user to include the payment.
007 Austin Code Legal Officer/reviewBSCPacketAndForwardToLegalOfficer
The Chain of Command (Austin Code Direct Chain Legal Office) reviews the case, examining the BSC folder, verifying labeling, signatures, and enters the case into a tracking BSCTrackingSheet (recording the address, who turned in the file, officer, officer's supervisors, whether it's an unsafe or dangerous structure, commercial or residential, date provided, and the date it was provided to the Legal Officer.) The Legal Office Chain of Command schedules a meeting to review the BSCCasePacket (ensure completeness, history of case, any contact with owner, dangerous, recommendation, substandard, length of issue, etc.). The meeting includes Legal Office Chain of Command, Chain of Command (Division Manager), Austin Code Legal Officer, Austin Code Officer, Chain of Command (Austin Code Officer's), BSC Coordinator.
008 Austin Code Legal Officer/reviewBSCPacket
The Austin Code Legal chain of command hands the file to the Austin code legal officer who reviews the case, examining the BCS folder, verifying the legal check list (type of structure, occupancy, reason for BSC action [ condition, circumstance, required, immediate escalation of enforcement action], Owner has not complied, owner unable to achieve compliance within time frame, provided NOV or within an approved extension time frame, appeal has been filed, relief sought in appeal, Administratively by field staff, field staff recommendations remedy [repair, vacate, secure or fence, demolition, historic preservation office notified (demos only), printouts of current case history with last action/event including contact or attempted contact with owner to inform them of case escalation to BSC (Secretary of state, utilities, homestead exemption, TCAD, title search, OPR, property tax search), and whether a title search has been completed, photocopy of delivered NoticeOfViolation with scanned copy of certified green card or photocopy of returned not delivered NOV with photo of on site posting, documentation in file identifying property owner(s), active permits (Building permit, plumbing permit, electrical permit, mechanical permit, demolition permit) labeled color photos for sited violations, labeled color photos of front/street view, photo of NOV posted at site, GIS and Structure Maps. Additionally, the BSCPacketChecklist is reviewed and printed and signed name by assistant division manager (chain of command).
009 Chain of Command/coordinateBSCCaseReview
The Chain of Command (Assistant Division Manager) schedules a BSCCase review with the Austin Code Officer, Investigator, Assistant Division Manager, Austin Code Legal Coordinator. The Chain of Command ensures the Checklist is complete, and any additional information necessary for BSCCase.
