Process Court Packet
Use Case Model

Scope: The Austin Code Officer submits the CourtPacket to the Austin Code Legal Coordinator to process the CourtPacket for the Court, Municipal or District. The Austin Code Legal Coordinator reviews the CourtPacket. If changes are required, the Austin Code Legal Coordinator requests the Austin Code Officer update the CourtPacket and return the CourtPacket when complete. If the changes to the CourtPacket involve modifying the Affidavit, then the use case ends and the Austin Code Officer must have the Affidavit notarized again. Once the CourtPacket is approved by the Austin Code Legal Coordinator, the Austin Code Legal Coordinator submits the CourtPacket to the Court. The Court reviews the CourtPacket and determines whether to create a CourtCase or dismiss the case. If dismissed, the Austin Code Legal Coordinator closes the LegalCase, and the use case ends. If a case is created, and the case is for Municipal Court, the Municipal Court uses JEMS to create a CourtCase. JEMS creates a CauseNumber for the CourtCase. The Austin Code Legal Coordinator then uses JEMS to identify cases on the Docket. Alternatively, the Case Management System requests JEMS identify the cases on the Docket. If a case is created, and the case is for District Court, the Austin Code Legal Coordinator requests the District Court provide the Docket and CourtCases. The Austin Code Legal Coordinator uses the Case Management System to update the LegalCase with the Docket information, CauseNumber, and CourtCase.

Process Court Packet
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Austin Code Legal Coordinator/processCourtPacket
The Austin Code Legal Coordinator reviews the CourtPacket and checks the CourtPacket for completeness and accuracy. If changes are required, the Austin Code Legal Coordinator requests the Austin Code Officer update the CourtPacket as necessary. If the Affidavit is modified, the Affidavit must be notarized again.
002 Austin Code Officer/updateCourtPacket
The Austin Code Legal Coordinator requests the Austin Code Officer update the CourtPacket for resubmission. The Austin Code Officer returns the updated CourtPacket to the Austin Code Legal Coordinator. If the Affidavit, included in the CourtPacket, requires changes, then the Affidavit must be notarized again and the Process Court Packet use case restarted.
003 Court/processCourtPacket
The Austin Code Legal Coordinator submits the CourtPacket to the Court, Municipal or District, and requests the Court add a CourtCase to the appropriate Docket. The Austin Code Legal Coordinator creates a copy of the CourtPacket, along with a copy from the Court, and stores it at Austin Code. The Court reviews the CourtPacket, and determines whether to create a CourtCase or dismiss the case.
004 Case Management System/closeLegalCase
The user uses the system to close the LegalCase. LegalCases are only closed once the Penalty has been paid for the LegalCase, or if the case was dismissed.
005 JEMS/createCourtCase
The Municipal Court creates a CourtCase, and a CauseNumber is generated in JEMS.
006 JEMS/identifyCasesOnDocket
The Austin Code Legal Coordinator uses the CourtPacket and requests JEMS identify the CourtCases on the requested Docket. JEMS returns the CauseNumber of the CourtCases. Alternatively, the Case Management System requests JEMS to identify the CourtCase automatically.
007 District Court/identifyCasesOnDocket
The Austin Code Legal Coordinator requests the District Court identify the cases on its Docket. The Austin Code Legal Coordinator provides information from the CourtPacket to help identify the CourtCases.
008 Case Management System/updateLegalCase
The Austin Code Legal Coordinator accesses the system and updates the relevant case with updates or follow ups from the BSC hearing (BSCDetermination, Order, etc.). The Austin Code Legal Coordinator updates the LegalCase to include the CauseNumber and final ruling of the case. The Austin Code Administrative Hearing Coordinator updates the LegalCase with the date of the hearing, outcome of hearing, penalty, isComplianceHearingNeeded, if penalty was paid, if penalty needs to be billed, and if the Violator came into compliance.
