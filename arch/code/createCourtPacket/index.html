Create Court Packet
Use Case Model

Scope: The Austin Code Officer creates an Affidavit and then assembles a CourtPacket. The Austin Code Officer then creates a LegalCase in the Case Management System. Alternatively, the Case Management System creates the Affidavit, per the Austin Code Officer's request, the system also creates the CourtPacket, and the LegalCase. The Austin Code Officer requests the Chain of Command review the CourtPacket. The Chain of Command reviews the CourtPacket and if changes are required, sends notes back on the requested changes. If approved, the Chain of Command signs the MunicipalCourtChecklist. The Chain of Command returns the CourtPacket to the Austin Code Officer. If the Case Management System was used to generate the court documents, then the Chain of Command accesses the CourtPacket in the Case Management System to provide notes and their approval. The process of creating the CourtPacket and requesting approval from the Chain of Command loops until the CourtPacket is approved. The Austin Code Officer requests a Notary notarize the Affidavit.

Create Court Packet
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Austin Code Officer/createAffidavit
The Austin Code Officer creates an Affidavit (either search warrant, or event/probable cause), using CodeCase information.
002 Austin Code Officer/createCourtPacket
The Austin Code Officer creates the CourtPacket which includes the Affidavit, CodeViolationCase (for supporting case history, Photographs, Violations), NoticeOfViolation, and Property (TCAD/WCAD information, Owner, etc.).
003 Case Management System/createLegalCase
The user accesses the Case Management System and creates a LegalCase. The user identifies the case type (Admin Hearing, Building Standards Commission, Municipal Court, or District Court).
004 Case Management System/createAffidavit
The user creates an Affidavit in the system (either search warrant, or event/probable cause), using CodeCase information.
005 Case Management System/createCourtPacket
The user accesses the system and creates an Affidavit. The user provides the Affidavit, CodeViolationCase (for supporting case history, Photographs, Violations), NoticeOfViolation, and Property (TCAD/WCAD information, Owner, etc.)
006 Chain of Command/approveCourtPacket
The Austin Code Officer delivers the CourtPacket to the Chain of Command and requests the Chain of Command reviews and approves the CourtPacket. If changes are required the Chain of Command doesn't sign the Checklist and provides notes on changes required. If approved, the Chain of Command signs the Checklist. The Chain of Command returns the CourtPacket to the Austin Code Officer.
007 Case Management System/approveCourtPacket
The user accesses the system and reviews the CourtPacket in the Case Management System. If changes are required the user updates the CourtPacket with notes and the CourtPacket is returned to the Austin Code Officer. If no changes are required the user approves the CourtPacket.
008 Notary/notarizeAffidavit
The Austin Code Officer requests a Notary within the Austin Code Department to notarize the Affidavit. The Notary notarizes the Affidavit and returns to the Austin Code Officer.
