Inspect Property Complaint
Use Case Model

Scope:

A Complainant contacts Austin 311 to report a CodeComplaint. Austin 311 creates a ServiceRequestAustinCode which ARR Customer Service uses to create a CodeComplaintCase in the Case Management System. Alternatively, Austin Code Officers report Violations directly to ARR Customer Service which results in a ServiceRequestAustinCode and later CodeComplaintCase. The CodeComplaintCase is updated with the AssignedCodeOfficer and AssignedProgramSpecialist.

After the CodeComplaintCase is created the Austin Code Officer inspects the Property per the Inspect Code Complaint Case use case. The Austin Code Officer contacts the Complainant as necessary for information, and if no Violation is found after the Inspection the CodeComplaintCase is closed. DoorHangers are placed, government entities contacted, and Inspections are performed until a Violation is either determined to exist, or not exist.

If a Violation exists the Create Code Violation Case use case starts. The Austin Code Officer creates a CodeViolationCase in the Case Management System, and performs different Inspections per the case disposition, including inspecting zoning land use violations, structure code violations. If emergency response is required the Engage Emergency Response use case is started. The Austin Code Officer creates additional CodeComplaintCases is new Violations are identified for non-compliant Properties. If the Property has multiple Violations the Austin Code Officer starts the Inspect Repeat Offender use case. The Austin Code Officer may issue Warnings and Citations. A NoticeOfViolation is sent out per the Send Notice of Violation use case. Community assistance is provided, if needed. Finally, the Complainant is contacted with the findings from the Inspection.

Once a CodeViolationCase has been created the Austin Code Officer has multiple routes to prompt resolution of the Violation through the Escalate Code Violation Case use case. The Austin Code Officer can extend the resolution time (Extend Violation Resolution Time use case), if requested. The Austin Code Officer performs an Inspection and identifies the appropriate escalation option: Citation (Issue Violation Citation use case), Building Standards Commission Review (Develop Building Standards Commission Case use case), preliminary hearing (Conduct Administrative Hearing use case), abate the Property (Create Abatement List use case), or Municipal/District Court action is necessary (Initiate Court Action use case). The CodeViolationCase is updated with the actions taken.

The Escalate Code Violation Case use case repeats on a set interval per the ViolationResolutionTime for the Violation. Each time the use case starts the Austin Code Officer determines the action required. A CodeViolationCase may have multiple escalation paths in motion simultaneously. A CodeViolationCase may have the same escalation option used multiple times. I.e. multiple Citations issued per the Issue Violation Citation use case.

The Inspect Property Complaint use case ends when the Violation is resolved.
