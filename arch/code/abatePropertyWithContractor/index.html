Abate Property With Contractor
Use Case Model

Scope:If the Property to abate is a transient camp or occupied the Austin Code Officer requests assistance with the cleanup (APD District Representative, Other Agencies, and Property Abator). During the cleanup multiple roles are involved in parallel. If the work to be completed doesn’t match the WorkRequiredList the Property Abator initiates the Approve Abatement List Adjustment use case for the necessary authorization to continue the abatement. The Austin Code Officer supervises the cleanup. The APD District Representative supervises the cleanup, primarily during transient camp abatement. Other Agencies supervise and assist with abatement as necessary specific to the agency’s scope.
If the Property to abate does not include a transient camp or occupied property (typically public Property, right of way, etc.) the Property Abator abates the property. If the work to be completed doesn’t match the WorkRequiredList the Property Abator initiates the Approve Abatement List Adjustment use case for the necessary authorization to continue the abatement. After the abatement is completed the Austin Code Program Specialist requests the Austin Code Officer verify the Property was abated.

The Property Abator, if abatement was completed by a Contractor, creates an Invoice. The Austin Code Officer then updates the CodeViolationCase with the results of the Property abatement.

Abate Property With Contractor
Sequence Diagram

Req# Type/Interface - Requirement Description Performance
001 Austin Code Officer/requestExternalAssistanceWithCleanup
The Austin Code Officer requests external assistance with transient camp cleanup and coordinates clean up date. Austin Code Officer contacts APD District Representative (phone or email). Additionally, the Austin Code Officer contacts additional external resources for assistance (Animal Control, Environmental, Watershed, Capitol Metro, etc.). Contractors are provided a WorkRequiredList identifying the type of abatement required.
002 Property Abator/abateProperty
The Contractor abates the Property. The Contractor takes photographs of before and after abatement. Camera provided by Austin Code Compliance to ensure DateStamp on Photograph. The Contractor provides DumpTickets (to verify lawful disposal of items at a dump, includes weight of items dumped).
003 Austin Code Officer/superviseCleanup
The Austin Code Officer supervises the cleanup effort. The Austin Code Officer is available to answer questions as needed. Additionally, the Austin Code Officer takes Photographs of the cleanup for addition to the CodeViolationCase.
004 APD District Representative/superviseCleanup
The APD District Representative supervises the cleanup to control public access and safety.
005 Other Agencies/superviseAndAbateCleanup
Other Agencies are on-site during the cleanup to supervise and provide abatement per their specialization.
006 Austin Code Officer/verifyPropertyAbated
The Austin Code Program Specialist requests the Austin Code Officer verifies the Property was abated. The Austin Code Officer reviews the Property and takes Photographs of the Abatement.
007 Property Abator/createInvoice
The Property Abator creates an Invoice which includes the following information: workDescription, workType, amountDue, dateCompleted, address, isApproved, and codeOfficerSignature upon completion of work.
008 Case Management System/updateCodeViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase. The Austin Code Officer updates the CodeViolationCase with information concerning Violation extensions, written request, reason for approval/denial, etc.
