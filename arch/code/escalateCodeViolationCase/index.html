Escalate Code Violation Case
Use Case Model

Scope: Code Violation Cases are reviewed and escalated continually until the Violation is resolved. The Responsible Party may request a Violation resolution extension that triggers the Extend Violation Resolution Time use case. If the Responsible Party does not request an extension, the Austin Code Officer uses the Case Management System to identify Property information to ensure it hasn't changed. The Austin Code Officer then performs an Inspection to see if the Violation is still unresolved. If the Violation is resolved, the Austin Code Officer updates the CodeViolationCase with notes on the resolution, closes the CodeViolationCase, and then closes the CodeComplaintCase per the Close Code Complaint Case use case. If the Violation is not resolved, the Austin Code Officer initiates an appropriate escalation use case per the CodeViolationCase disposition:

  • If the Violation is not in compliance, the Austin Code Officer begins the Issue Violation Citation use case.
  • If the Violation is not in compliance for a structural issue/dangerous condition, the Develop Building Standards Commission Case use case begins.
  • If a preliminary hearing is required, the Conduct Administrative Hearing use case occurs.
  • If the Violation is not in compliance and property abatement is required, the Create Abatement List use case is started by the Austin Code Officer to abate the property.
  • If District Court is necessary, the Austin Code Officer starts the Initiate Court Action use case.

The Austin Code Officer uses the Case Management System to update the CodeViolationCase with notes on the escalation actions taken regarding the CodeViolationCase.

Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Case Management System/identifyPropertyInformation
The Austin Code Officer requests the Case Management System to look up certificate of occupancy, site plan, building permits, electrical permits, plumbing permits, mechanical permits, other permits, change of use, plan review, variances, neighborhood plans, conditional overlays, licensing, and CaseHistory.
002 Austin Code Officer/performInspection
If the violation concerns hazardous conditions, the Austin Code Officer applies Personal Protective Equipment (PPE). The Austin Code Officer inspects the Property. The Austin Code Officer inspects from the exterior (initially from the Right of Way) to the interior (if the interior is accessible and permission has been granted by legally interested party or someone in Care/Control of Property). The Austin Code Officer take Photographs and Measurements (hot water temperature, air quality, interior temperature, light meter, ground-fault circuit interrupter, gas meter, carbon monoxide, smoke detector, fences, grass height, driveway and walkways, check impervious cover, etc.) as needed. As needed, the Austin Code Officer performs research to identify the Property Owner, Offense, ViolationType, OwnerInformation, and relevant PropertyInformation (GIS system, ARR Site, Annexation Information, Amanda Permit History, Site Plan, TCAD/WCAD for OwnerHistory and DeedHistory, CC&B, ZoningHistory, CodeOfOrdinances, etc.). The Austin Code Officer interviews relevant parties. For Waste Hauler Inspection: The Austin Code Officer makes contact with driver, visually inspects vehicle for decals on door, asks driver about awareness requirement for waste hauler license, ensures driver is operating within City of Austin (outside of city exempt), double check the Waste Hauler license, check for safety equipment violations, driver's license of driver, and take photos of truck (including both doors and front license plate). If a short-term rental complaint, the Austin Code Officer responds at a faster time than other complaints. On Fridays and Saturdays, the Austin Code Officer performs inspection during day, ands from 6:00 P.M. - 2:00 A.M. On Sundays, inspections occur from 8:00 A.M. - 12:00 P.M.
003 Case Management System/updateCodeViolationCase
The user updates the CodeViolationCase with relevant information to the case. If the CodeViolationCase generated a StopWorkOrder, the StopWorkOrder is attached to the CodeViolationCase. The user attaches a copy of the DiagramOfWorkRequired. If a board and secure occurs, a BoardAndSecureBill is attached to the CodeViolationCase by Austin Code Admin. If a sign was removed, the Austin Code Officer adds the sign and information to the CodeViolationCase. Additionally, if CautionBarriers were placed, information on the CautionBarriers is added to the CodeViolationCase. If property information was updated, the Case Management System includes updated property information. The user updates the CodeViolationCase with information relevant to the change. This includes, but is not limited to, BoardAndSecureBill, PropertyAbatementBill, FollowUpNotes, photographs (boarded, non-boarded, posted notifications, placards), emails, PhoneCallSummary, other relevant information and inter-department communications. If a ServiceCase is created, the Public Information Office Admin adds notes to the CodeViolationCase including the CitizenNeed and ServicesRecommended for ServiceCase. If CorrectiveResources are identified, the user updates the CodeViolationCase with a list of CorrectiveResources provided (three resources) and notes of contact with Owner/Tenant. HardshipReason is added to the CodeViolationCase. If new violations are identified, the user adds the new violation to the Case Management System and the violation is attached to the original CodeComplaintCase. If a TitleSearch is completed, the TitleSearch is added to the CodeViolationCase. If additional LegallyInterestedParties are identified, the parties are added to the CodeViolationCase. If a NoticeOfViolation is generated, the NoticeOfViolation is added to the CodeViolationCase. If posted at the property, photographs are attached to the CodeViolationCase. If the NoticeOfViolation is delivered and a certified mail receipt is received, the receipt is added to the CodeViolationCase. If a citation is created, the Case Management System attaches the citation to the CodeViolationCase. Additionally, the user scans and uploads a copy of the physical citation to the Case Management System. If an AdministrativeHearingWarning is issued, the user updates the CodeViolationCase with information on the issued AdministrativeHearingWarning. If an AdministrativeHearingCitation is issued, the user updates the CodeViolationCase with the AdministrativeHearingCitation, and a LegalCase is created. The user scans and uploads returned liens to the CodeViolationCase. If a lien is filed for the property, the TRVNumber is added to the CodeViolationCase. The Austin Code Officer updates the CodeViolationCase with information concerning Violation extensions, written request, reason for approval/denial, etc.
004 Case Management System/closeCodeViolationCase
The CodeViolationCase is closed by a user or the system once all Violations have been resolved.
