Manage Contractor Request
Use Case Model

Scope:

The Contractor accesses the system , and chooses to send a Request for Information or chooses to select an existing Request for information and add a response. The System assigns a sequential unique number if this is a new Request, and provides a screen for the contractor to specify their inquiry. The system tracks all relevant information like date submitted, submitted by whom, Date responded, date resubmitted. The system automatically notifies the Inspector, Inspection Supervisor, Project Manager, Sponsor, Inspection field Engineer, and Consultant that a Request has been initiated. Each may provide a comment via the system, until the Project Manager deems the answer complete. At this point, the System records the date the response is issued, and sends notification to the Contractor indicating their requested information is now available on the system.

Manage Contractor Request Interface Descriptions

Type/InterfaceDescription
Administer Capital Construction Data/
createOrLookupRequestForInformation		 The Contractor Logs into the system and uses the CIP no. to locate their project. If they are looking up an existing Request for information, the system will prompt them for the unique Request for information number, or will provide a selection of the available Requests For Information already associated. The Contractor either inputs the new request, modifies the existing request with additional questions, or resubmits an existing request with clarification added. At this time, the system provides the capability to upload documents as part of the request. The System requires the user to write a description of their request . The System Captures the date and time the request is submitted.
Administer Capital Construction Data/
notifyRespondersOfPendingRequestForInformation		 The System sends a notification to the Responders and the Project Manager indicating a Request for Information has been added to the project, and that a response is necessary.
Administer Capital Construction Data/
addCommentsToRequestForInformation		 The Responders access the system, and review the Request for Information, adding their comments as appropriate.
Administer Capital Construction Data/
EvaluateAndConsolidateResponderInput		 The Project Manager accesses the system to evaluate the responses. The Project Manager can Suppress comments provided by the Responders. The Project Manager will edit and or/determine which comments can be seen by the contractor, and will use the system to close the response ability when they deem appropriate. The Project Manager may also attach documents as part of the response. In which case these attachments are visible by the contractor and Responders, once the response has been approved. After this time, no others can enter information related to the Request for information besides the Project Manager. Additionally the Project Manager may unlock the response ability should the need occur.
Administer Capital Construction Data/
markCompleteAndNotifySubmitter		 The Project Manager uses the System to indicate the Response is ready for the contractor. The System records the date time the Request for Information is completed. The System will send a notification to the Contractor and all responders, listing the CIP No, the Request for information unique Id, the date submitted as well as the date replied.
