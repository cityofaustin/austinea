Publish Daily Report
Use Case

Use Case Scope

Populate a standardized form that captures project data in the form of weather, equipment inventory, number of workers, visitors, test results data and descriptive text. The Form is filled out by the Inspector and vetted by Supervisors, Field Engineers, Engineering Technicians and Administrative Assistants.

Publish Daily Report Interface Descriptions

Type/InterfaceDescription
Inspector/
collectProjectDocumentation		 The Inspector collects daily project data , [Header Info] such as Name of Project, task order, day of week, date, day range (i.e. day 1 of 6, etc.), start date, substantial completion date, CIP sub ID numbers, contractor, superintendent, job ID number, to-from date range, total contract days (whether working or calendar), table of equipment (make, model, and serial number, and type of equipment Enumeration: A: dozer B: backhoe C: trackhoe D: front end loader E: blade F: milling machine G: pneumatic roller H: steel wheel roller I: paving machine J: boring machine K: trencher L: pick up truck M: compactor N: water truck O, P, Q, R: open for other things Table of construction permits: type of permit, date permit issued, listing of dump sites, list of permits (planning permit [from planning department], street cut permit, water/wastewater permit), water and wastewater testing number List of approved sub-contractors [Beginning of report body] Count of how many are on site (superintendent, safety rep, foreman, operators, laborers), note weather, high temp, low temp, time work started, time work ended, if shoring was used and type of shoring, check traffic control (AM & PM), check environmental controls. Verify that traffic controls are set (yes or no). In the body of the daily report there is an itemized list providing pay item, quantity, description of work, location or station. Additionally, there is an itemized equipment list providing the equipment letter, hours in use, and hours in standby. A testing section contains an itemized list of types of test performed, location/station of test, results of test, pass or fail for test(s). The overall daily report contains a remarks section which is a written account of verbal instructions, problems, work delayed and reason, utility relocation work or conflicts, miscellaneous remarks pertinent to work, and visitors to project. Lastly there are two signature blocks: one for the Inspector, and one for the Inspection Supervisor. See Logical Data Model for items described above
Administer Capital Constuction Data/
addDailyReportInfoToProject		 The Inspector accesses the system, specifies the project, selects a day or date range, and adds Daily Report information as necessary.
Administer Capital Constuction Data/
recordTestResults		 The Inspector indicates one of the uploaded documents is a test result. The system offers a different screen requesting Pass/Fail and metric information about the specific test. Additionally the Inspector indicates WHICH test was conducted, date test was conducted, and by whom. The system offers the inspector a radio box to check indicating "Back Charge Contractor", and another radio box to indicate "Retesting Is Required" The system sends notifications to the Project Manager and Inspector / Inspector Supervisor indicating the test results were added.
Supervisor/
reviewAndApproveDailyReport		 The Supervisor verifies that Daily Report is completed (all fields are filled out). Supervisor checks for completeness, public appropriate text, and factual information instead of opinion. If there are no problems the Supervisor will approve the Daily Report effectively "locking" it for any modifications. In instances where changes are needed the Supervisor, or one of their delegates, can "unlock" a Daily Report for modification.
Administer Capital Constuction Data/
reviewDailyReport		 The user accesses the system, specifies the project, selects a day or date range, and reviews Daily Report information as necessary.
Inspector/
correctReportIssues		 The Inspector is notified by the Inspection Supervisor of issues with the Daily Report. This indicates the Supervisor has "unlocked" the Daily Report for modifications by the Inspector. The Inspector modifies the Daily Report accordingly. Upon resubmission the Daily Report will once again become locked. The system records the date, time, and user ID of the Inspector when a change is affected on the Daily Reports.
Administer Capital Constuction Data/
updateDailyReportInfo		 The user accesses the system, specifies the project, selects a day or date range, and updates Daily Report information as necessary.
Field Engineer/
spotCheckSectionOfDailyReport		 The Field Engineer selects from the stack of Daily Reports. The Field Engineer randomly selects one Daily Report for review and then verifies Daily Report. The Field Engineer accesses the system and verifies the required daily reports have been posted as expected. The Field Engineer spot checks Daily Report for factual information instead of opinion, and Daily Report is filled out completely.
Administrative Assistant/
performDocumentControlActivity		 The Administrative Assistant takes all of the Daily Progress Reports for a specific date range and verifies completeness (not missing a day, approved).
Administer Capital Constuction Data/
performCompletenessCheck		 The Engineering Technician verifies the Daily Progress Reports are present and accurate, as the project is now completed.
Contractor/
deliverRequestedReportToClient		 The user accesses the system, specifies the project, selects a day or date range, and an email address. The system prompts for the output name and location of the report to be generated and then generates a PDF formatted report at the desired location.
