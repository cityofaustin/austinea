Configure Report
Use Case Model

Scope: A CRM User accesses the system and creates a ReportChange. The system updates the WorkQueue for the assigned Report Manager from the ReportChange. The Report Manager accesses the system and queries their WorkQueue. The Report Manager creates or updates the requested Report, and requests the system publishes the Report to the desired CRMSystem. If a ReportSchedule is required to automate running or sending the Report, the Report Manager creates or updates the appropriate ReportSchedule. The Report Manager updates the ReportChange with information on the work completed and the system updates the WorkQueue for the CRM User responsible for reviewing the Report.

Configure Report
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Provide Customer Relationship Management/createReportChangeRequest
A CRM User submits a ReportChange in the system. The CRM User provides report name, description of changes, reason for change, Report Manager to complete the work, Point of Contact. The system updates the WorkQueue for the identified Report Manager.
002 Provide Customer Relationship Management/updateWorkQueue
The system updates the CRM User's WorkQueue to include the KnowledgeBaseItem's requirement for acknowledgment. CRM User accesses the system and updates the WorkQueue to identify acknowledgment of KnowledgeBaseItem. After completed the CRM User's WorkQueue removes the KnowledgeBaseItem from the WorkQueue. The system updates the CRM User's WorkQueue indicating a KnowledgeBaseItem did not index properly, and requires CRM User review. The system updates the Editor's (CRM User) WorkQueue indicating a KnowledgeBaseItem received a Comment and requires CRM User review.
003 Provide Customer Relationship Management/queryWorkQueue
A CRM User requests the system queries a WorkQueue. The CRM User provides a variety of information to identify a WorkQueue, or subset of WorkQueue including: ServiceRequest information (CRMSearch, i.e. Status, Type, Date/Time, Outcome, Activity, Question/Answers, Location, GIS Information, Service Level Agreement,), and any ServiceRequest information is available to use for query. Queries are sortable by CRM User for any criteria provided.
004 Provide Customer Relationship Management/createUpdateReport
The Report Manager uses the system to create or update the requested Report per the ReportChange.
005 Provide Customer Relationship Management/publishReport
The Report Manager requests the system publishes the Report to the desired CRMSystem.
006 Provide Customer Relationship Management/createUpdateReportSchedule
The Report Manager creates or updates the ReportSchedule. The ReportSchedule identifies the Report, recurrence, start time, stop time, file format, delivery method, and recipients.
007 Provide Customer Relationship Management/updateReportChangeRequest
The Report Manager updates the ReportChange with notes on the work completed and the system updates the WorkQueue for the point of contact to review.
