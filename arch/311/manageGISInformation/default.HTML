Manage GIS Information
Use Case Model

Scope: The GIS Manager requests the system create or update a GISLayer. The GIS Manager publishes the updated GISLayer to the desired system. Alternatively, the system’s GIS information is updated by an automated process, either an external GIS system or the Enterprise Service Bus. The system updates each GISLayer per the update schedule in the GISLayer. If GISLayers are updated through external GIS the system requests the external GIS system to update the GISLayer. GIS returns the updated GISLayer. If GISLayers are updated through the Enterprise Service Bus the system creates a COAESBMessage and then requests the Enterprise Service Bus update the GISLayer. The Enterprise Service Bus returns the updated GISLayer. The system sends notification to the GIS Manager of update. If GISLayer testing is required the GIS Manager tests the GISLayer and then publishes the GISLayer to the desired system. If GISLayer doesn’t require testing the system publishes the GISLayer.

Manage GIS Information
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Provide Customer Relationship Management/createUpdateGISLayer
GIS Manager accesses the system and creates or updates a GISLayer. GISLayers support include line, point, polygon, other shape file. The GIS Manager defines GISLayer information. If updating a GISLayer the system maintains an archived version of the GISLayer for auditing purposes. Alternatively, the system performs the update GISLayer action automatically based on an update/refresh schedule defined by the GIS Manager. Automatic updates are completed by connecting to external GIS systems or the Enterprise Service Bus.
002 Provide Customer Relationship Management/publishGISLayer
The GIS Manager requests the system publishes the GISLayer to the destination CRM system (training, production, etc.). The GIS Manager can select historical GISLayers for publishing if desired.
003 GIS/updateGISLayer
The system requests an external GIS system (ArcGISOnline, City of Austin GIS, Google, etc.) provides a GISLayer update. The external GIS system returns the updated GISLayer.
004 Provide Customer Relationship Management/createCOAESBMessage
The system creates a COAESBMessage. The system creates a COAESBMessageHeader and adds the InterfaceServiceRequest, CRMDisposition, Resource, etc. to the message body.
005 Enterprise Service Bus/updateGISLayer
The system requests the Enterprise Service Bus updates the GISLayer. The Enterprise Service Bus contacts the GIS system identified in the COAESBMessage and returns the updated GISLayer.
006 Provide Customer Relationship Management/sendNotification
The system sends a Notification (email, text, etc.) to the recipient. Notification includes Attachments as relevant. If a response from a Citizen is requested the system provides either a URL to respond, generates a subject line for response to the system so the system recognizes the response, etc. The system saves a copy of the Notification after transmission for archival purposes (copy of message, recipients, transmission date/time, schedule, etc.). For public notifications the system sends the Notification per the schedule defined by the Public Information Manager. For ServiceRequest Notifications the system attaches a copy of the Notification with the ServiceRequest for archival purposes. CRM Users resend/cancel Notifications as desired. Notifications are sent ad hoc and also scheduled based on configuration by a CRM Manager. If a delivery receipt is configured the system tracks delivery status of the Notification. For GISLayer updates the system notifies the GISManager identified for the GISLayer. The system includes the number of changes to the GISLayers, date/time of retrieval from GIS, and GISLayer name. For InterfaceServiceRequests, if the system receives an EnterpriseServiceBusStatus back that is not success then the system notifies the CRM Manager. If a KnowledgeBaseItem fails to index the system notifies the KnowledgeBaseItem's Editor with the KnowledgeBaseItem's title and date of failed index.
007 GIS Manager/testGISChanges
The GIS Manager tests the GISLayer changes to ensure accuracy and validity of data. GIS Manager either approves or rejects the changes. If approved the GIS Manager publishes the GISLayer changes.
