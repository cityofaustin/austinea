Transmit Amanda Data to ESB
Use Case Model

Scope: When a Case's Status is updated, and the Case interfaces with another system, the Transmit Amanda Data to ESB use case is triggered. Status changes occur when a Case is originally created (null to new status), when an Inspector updates the Status of the Case, or if a child Case (CodeViolation, etc.) has its Status updated and the parent Case's Status is updated by Amanda. Alternatively, the Transmit Amanda Data to ESB use case is triggered when a 311Feedback AttemptResult is created. The Amanda system creates a COAESBMessage which contains the updates to send to the desired system (CSR). The COAESBMessage contains: assigned Inspector name and phone number, FolderRSN, Status, AttemptResult name, and AttemptResult notes. Once the COAESBMessage is created the Amanda system requests the Enterprise Service Bus update a ServiceRequest. The destination system for the update is defined in he COAESBMessage.
