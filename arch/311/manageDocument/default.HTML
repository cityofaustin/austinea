Manage Document
Use Case Model

Scope: A Document Author creates/updates a Document (QMS, ReferenceMaterial, etc.) and sends to a Document Uploader. The Document Uploader uploads the Document to the system (CSR, SharePoint, network location, etc.) and provides: document information, whether training cases are needed, etc. After the Document is added/updated in the system the Document is routed to a Document Reviewer. The Document Reviewer reviews the Document. If changes are necessary the Document is sent back to the Author for revisions. If no changes are necessary, and the document is not QMS related, the Document Reviewer sends the Document to the Document Approver (often the same staff member) for publishing. If no changes are necessary, and the Document is QMS related, the Document Reviewer sends the Document to the Document Approver. The Document Approver reviews the Document. If changes are necessary the Document is sent back to the Author for revisions. The Author completes revisions and sends back to the Document Reviewer. If the Document change is deemed major (new document, change to process), the Train Staff on QMS Compliance use case is initiated by the Document Approver. If the Document change is deemed minor (formatting, spelling, non-process change) training is not required.  The Document Approver publishes the Document in the system. If the Document is located on SharePoint or a network location, the Document Approver sends Notification of the Document to appropriate staff.

Manage Document
Activity Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Author/Create or Update ISO Document and Upload to System
A Document Author creates or updates a Document. If creating a new QMSDocument the QMSProcess initiates the document creation. The Author uploads the Document to the system (CSR, SharePoint, Network location, etc.), enters document information, and routes the Document to the Document Reviewer responsible for that particular Process, or QMSProcess.
002 Document Uploader/Upload Document
The Document Uploader uploads the Document to the system (CSR, SharePoint, Network location, etc.), enters document information, and routes the Document to the Document Reviewer responsible for that particular Process, or QMSProcess.
003 Document Reviewer/Review Document and Route to Appropriate Staff
The Document Reviewer checks the Document for spelling, grammar, clarity, context, content accuracy, format (template, header, footer, file type, etc.), consistency, referenced information, keywords, search depth, etc. If revisions are required, the Document is returned to the Author for revisions. The Document Reviewer enters notes on their decision. If no revisions are required, and the Document is a QMSDocument, the Document Reviewer sends the Document to the Document Approver. If no revisions are required, and the document is not a QMSDocument, the Document is forwarded to the Document Approver for publishing (often the same staff as the Document Reviewer).
004 Document Approver/Review Document and Take Next Actions
The Document Approver checks the Document for spelling, grammar, clarity, context, content accuracy, format (template, header, footer, file type, etc.), consistency, referenced information, keywords, search depth, Document Reviewer's notes, etc. If revisions are required, the Document is returned to the Author for revisions. The Document Approver enters notes on their review result, and determines whether training is required. If the Document change is considered major, or the Document is new, the Train Staff on QMS Compliance use case is started.

006 Document Approver/Publish Document
The Document Approver uses the system to publish the Document. Once published the Document is visible to staff. If the Document is located on SharePoint or a network location the Document Approver sends Notification of the Document to staff.
