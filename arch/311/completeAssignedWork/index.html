Complete Assigned Work
Use Case Model

Scope: The system provides multiple options and services to facilitate CRM Users completing their assigned work and closing a ServiceRequest:

  • If the ServiceRequest interfaces with an external system ServiceRequest updates are transmitted back from the Enterprise Service Bus to the system.
  • If the ServiceRequest is configured with auto-assignment rules for Resources or CRM Users the system updates the ServiceRequest with the appropriate assignment.
  • If the ServiceRequest is assigned to Resources manually by CRM Users the CRM User updates the ServiceRequest with the appropriate assignment. The system updates the WorkQueue for the assigned Resource once assigned, either manually or automatically.
  • If the ServiceRequest has notifications configured for transmission the system sends the notifications per configuration.
  • CRM Users either query their WorkQueue for their work including ServiceRequests, or utilize a Dashboard, configurable by the CRM Manager or CRM User, to view work.
  • If a WorkRoute is required the CRM User requests the system generates a WorkRoute for the requested ServiceRequests. The system provides different WorkRoute options including generating routes based on shortest distance traveled, etc.
  • If GISMaps are needed the CRM User requests the system generates the appropriate GISMap. GISMaps are configured by the CRM User to include different GISLayers, colors, transparency, etc.
  • If a ServiceRequestReport is required of the ServiceRequest the CRM User requests the system generates a ServiceRequestReport. ServiceRequestReports are customized by the CRM User to include portions of the ServiceRequest the CRM User requires to complete resolution of the request. Additionally, multiple reports may be printed simultaneously for WorkQueue ServiceRequests.

    The CRM User updates the ServiceRequest including adding comments and notes to Activities, uploading Attachments, answering Questions specific to resolving the ServiceRequest, etc.
  • If the ServiceRequest was configured to include triggers for additional processing the system updates the ServiceRequest as needed based on the triggers and resulting actions.
  • If follow on ServiceRequests are required the CRM User, or the system if ServiceRequest is configured, creates a new ServiceRequest and the two ServiceRequests are linked. Linked ServiceRequests are added to the RelatedServiceRequestList for both ServiceRequests.

    If the CRM User is mobile the system provides additional functionality:
  • If a WorkRoute was created the CRM User requests a WorkRoute. The WorkRoute is displayed along with relevant information. If a Traffic System is available to look up TrafficData the system requests the Traffic System return the data and the system updates the WorkRoute.
  • If the CRM User requires additional Activities the CRM User requests new Activities from the system. The system returns a list of available Activities for the CRM User to select and assign themselves to.
  • If the CRM User has arrived at the Location of the ServiceRequest the CRM User checks in and the system updates the ServiceRequest with their arrival time and Location based on GPS coordinates.
  • If nearby mobile coworkers are needed the CRM User requests the system display nearby coworkers. The system displays on a map the various nearby coworkers based on the CRM User's requirements. The CRM User updates the ServiceRequest when mobile.
  • If a WorkRoute was created for the CRM User the WorkRoute is updated each time the CRM User completes their assigned Activity for a ServiceRequest.

    Citizens look up the status of their ServiceRequest their an online website through the system. The Citizen provides a ServiceRequestNumber, along with their relevant Participant information and the system returns the ServiceRequest.

    Complete Assigned Work
    Sequence Diagram

    Req#Type/Interface - Requirement DescriptionPerformance
    001 Provide Customer Relationship Management/updateServiceRequest
    The system, or a CRM User, updates a ServiceRequest per the requested update. If the ServiceRequest is configured with auto-assignment rules for CRM Users the system assigns the ServiceRequest Activities to resources based on Questions/Answers, Location, GISLayer, Time of Day, Shift, Route, Zone, etc. encompassing business process rules for group. CRM Users update the ServiceRequest, and if interfaced with an external system, may request an update be sent to the interface. If an Interface updates the ServiceRequest the Interface transmits an update Notification to the system and includes the information to update/append, etc. Interface updates any portion of the ServiceRequest required to complete ServiceRequest (Status, Questions/Answers, Activities, etc.). ServiceRequest Activities are assignable to multiple Resources/CRMUsers and Activities track Resource usage. The ServiceRequest is configured to require specific be completed by a user before it may be saved (answering a question, completing an activity, etc.). ServiceRequest updates are completed by CRM Users and any portion of the ServiceRequest may be updated by the CRM User per their Access Permissions. Parts of the ServiceRequest (activities, questions, attachments, participants) are designated as required for completion and must be completed before the ServiceRequest is closed. As CRM Users check in at Locations the system updates Activities with arrival date/time, Location (GPS coordinates). Activities allow CRM Users to estimate completion duration and actual completion duration. Multiple activities may be added to a ServiceRequest simultaneously either by a CRM User, Interface, or auto-processing rules. CRM Users can attach relevant information to the ServiceRequest including (pointers to other ServiceRequests, Locations, GISData, etc.). The system updates the ServiceRequest with any Notifications sent. If Citizens provide a requested response to the system the system updates the ServiceRequest with the response Notification as an Attachment. Additionally, the system saves any Attachments included in the Notification. When the ServiceRequest work is completed the ServiceRequest status is changed to a Closed status. Once closed a ServiceRequest is not editable by CRM Users. Some CRM Users, based on their access permissions, may reopen a ServiceRequest to make additional edits.
    002 Provide Customer Relationship Management/updateWorkQueue
    The system updates the CRM User's WorkQueue to include the KnowledgeBaseItem's requirement for acknowledgment. CRM User accesses the system and updates the WorkQueue to identify acknowledgment of KnowledgeBaseItem. After completed the CRM User's WorkQueue removes the KnowledgeBaseItem from the WorkQueue. The system updates the CRM User's WorkQueue indicating a KnowledgeBaseItem did not index properly, and requires CRM User review. The system updates the Editor's (CRM User) WorkQueue indicating a KnowledgeBaseItem received a Comment and requires CRM User review.
    003 Provide Customer Relationship Management/sendNotification
    The system sends a Notification (email, text, etc.) to the recipient. Notification includes Attachments as relevant. If a response from a Citizen is requested the system provides either a URL to respond, generates a subject line for response to the system so the system recognizes the response, etc. The system saves a copy of the Notification after transmission for archival purposes (copy of message, recipients, transmission date/time, schedule, etc.). For public notifications the system sends the Notification per the schedule defined by the Public Information Manager. For ServiceRequest Notifications the system attaches a copy of the Notification with the ServiceRequest for archival purposes. CRM Users resend/cancel Notifications as desired. Notifications are sent ad hoc and also scheduled based on configuration by a CRM Manager. If a delivery receipt is configured the system tracks delivery status of the Notification. For GISLayer updates the system notifies the GISManager identified for the GISLayer. The system includes the number of changes to the GISLayers, date/time of retrieval from GIS, and GISLayer name. For InterfaceServiceRequests, if the system receives an EnterpriseServiceBusStatus back that is not success then the system notifies the CRM Manager. If a KnowledgeBaseItem fails to index the system notifies the KnowledgeBaseItem's Editor with the KnowledgeBaseItem's title and date of failed index.
    004 Provide Customer Relationship Management/queryWorkQueue
    A CRM User requests the system queries a WorkQueue. The CRM User provides a variety of information to identify a WorkQueue, or subset of WorkQueue including: ServiceRequest information (CRMSearch, i.e. Status, Type, Date/Time, Outcome, Activity, Question/Answers, Location, GIS Information, Service Level Agreement,), and any ServiceRequest information is available to use for query. Queries are sortable by CRM User for any criteria provided.
    005 Provide Customer Relationship Management/displayDashboard
    The CRM User requests the system display a Dashboard and provides a CRMSearch for the Dashboard criteria. The Dashboard is customized by the CRM User to include different query criteria: ServiceRequests, ServiceRequest statistics (number submitted, divided by group, overdue ServiceRequests, etc.), formatting, etc. The Dashboard is configurable with a refresh interval and automatically updates as the Dashboard remains active. Dashboards display graphs tables, and maps as configured by the CRM User. Dashboards may be saved, are sharable with CRM Users, and external parties through links.
    006 Provide Customer Relationship Management/createWorkRoute
    The CRM User requests the system creates a WorkRoute and provides a CRMSearch to define the WorkRoute's criteria. The CRM User provides query criteria for the WorkRoute including ServiceRequestType, Date(s), Location, User, Activity (type, due date, created date, interval till overdue), Status, Interval till overdue, Priority, Threshold to prioritize, WorkRoute optimization (shortest travel time, etc.), Shift, nearby WorkRoute range. WorkRoutes contain Location(s), ServiceRequest(s), Time (estimated activity start and end, travel time[?]), Route (stops, list of ServiceRequests), and nearby WorkRoutes.
    007 Traffic System/createWorkRoute
    The system requests the Traffic System provide real-time TrafficData for the included WorkRoute. The Traffic System returns the TrafficData to the system.
    008 Provide Customer Relationship Management/updateWorkRoute
    The system updates the WorkRoute to reflect updates to ServiceRequests (completed Activities, Status changes, new ServiceRequests, closed ServiceRequests, Outcomes), or to include TrafficData. The system redraws WorkRoutes as needed based on the initial WorkRoute optimization chosen by the CRM User when the WorkRoute was created (shortest distance, time, etc.). Updated WorkRoutes are presented to the CRM User when completed.
    009 Provide Customer Relationship Management/createGISMap
    The CRM User requests the system creates a GISMap based on various criteria from a CRMSearch (all ServiceRequest information is available for query). The CRM User defines a GIS Area and provides filters (non-spacial data, ServiceRequestType, Date(s), GISLayer(s), etc.), defines labels, transparency, overlays, BaseMap, etc. The system generates a GISMap that's available for saving, or attachment to ServiceRequests, etc.
    010 Provide Customer Relationship Management/createServiceRequestReport
    The CRM User requests the system prints a ServiceRequestReport. The ServiceRequestReport is customizable by the CRM User (Label, Title, Questions/Answers, Activities, Attachments, History, Participant, etc.). The CRM User prints a single ServiceRequestReport or prints multiple reports for different ServiceRequests based on their query. Reports are exportable to a variety of formats, can be saved, attached, and emailed. Alternatively, ServiceRequestReport is scheduled by the CRM User to run automatically and print.
    011 Provide Customer Relationship Management/createServiceRequest
    A CRM User accesses the system and requests the system creates a ServiceRequest. The system provides multiple ServiceRequestTypes to choose from, pre-configured by a CRM Manager. The system requests a Location, if required, and validates the Location against configured GISLayers (validate against GISLayers, street address or intersection required). The system performs GIS lookup, and general lookup for relevant information (nearby places, resources, schools, parks, alerts, other ServiceRequests, GIS information, KnowledgeBase items, etc.). The system provides Questions of various types, configured by CRM Manager, and prompts the user with Alerts (triggered by Question answers or selection of ServiceRequestType) to alter the ServiceRequest business flow as necessary. The system provides the ability to add, remove, Questions based on previous answers, and alter the business flow based on Question answers. The user is able to attach files to the ServiceRequest. The system sends messages (pages, emails, etc.) based on Question answers to configurable recipients. The system assigns a Priority and Status to the ServiceRequest based on the configured options by the CRM Manager. The system generates a ServiceRequestNumber (unique tracking number for the individual request) once the ServiceRequest is saved. The system allows the CRM user to capture Participant information (name, phone number, location, email, Questions, etc.). The system provides a field to enter general comments/details as needed. Additionally, the system allows users to perform a bulk ServiceRequest creation. The system completes any ServiceRequest configuration for the bulk created ServiceRequests similar to singularly created ServiceRequests. The system provides a mass entry intake option that presents specialized ServiceRequests. MassEntryServiceRequests are created in a user defined status, typically Closed. If the MassEntryServiceRequest is created in a Closed Status the system doeesn't create Activities for the work group to complete as the work was completed previously. The system provides a Website intake for ServiceRequests along with WebsiteServiceRequests that map back to a ServiceRequest in the system. The system provides a Mobile Application intake for ServiceRequests along with MobileApplicationServiceRequests that map back to a ServiceRequest in the system. The system provides the capability for an external system to interface with it. The system interfacing provides the InterfaceServiceRequest information and the system maps the InterfaceServiceRequest back to a ServiceRequest in the system. CRM Users create attachments to other data points in CRM including (ServiceRequests, Locations, GISInformation, etc.).
    012 Provide Customer Relationship Management/linkServiceRequest
    The system links a ServiceRequest to another ServiceRequest. Each ServiceRequest's RelatedServiceRequestList is updated to show the link between related ServiceRequests. ServiceRequests are linked either manually by CRM Users or through automated triggers configured in the system by the CRM Manager.
    013 Provide Customer Relationship Management/displayWorkRoute
    The CRM User requests the system displays the requested WorkRoute. The system displays the WorkRoute on the requested device.
    014 Traffic System/provideTrafficData
    The system requests the Traffic System provide real-time TrafficData for the included WorkRoute. The Traffic System returns the TrafficData to the system.
    015 Provide Customer Relationship Management/requestActivities
    The CRM User requests additional Activities from the system through a query. The system displays available Activities the CRM User is capable of completing resolution on (Location, Resource Availability). The CRM User selects the Activities they choose to work/assign to themselves.
    016 Provide Customer Relationship Management/checkInLocation
    The CRM User checks in at the Location. The system tracks the user's Location (GPS Coordinates, Address), Date/Time of Arrival, and updates the ServiceRequest with arrival information.
    017 Provide Customer Relationship Management/displayNearbyCoWorkers
    The CRM User requests the system displays nearby coworkers. The CRM User specifies the query including Location, distance, etc. and the system displays coworkers in a list and map. The system provides the CRM User the ability to contact coworkers as needed.
    018 Provide Customer Relationship Management/lookupServiceRequestOnWebsite
    Citizens look up the status of their ServiceRequest their an online website through the system. The Citizen provides a ServiceRequestNumber or CRMSearch, along with their relevant Participant information and the system returns the ServiceRequest.
