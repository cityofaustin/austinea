Create Case
Use Case Model

Scope: The Enterprise Service Bus requests the Amanda system create a Case (complaint, short term rental license, etc. based on disposition of the message). The Amanda system creates the Case. During the Case creation process the Amanda system updates the Info Tab's fields with information from the original message.

The PropertyRSN is then looked up:

  • If an interface to GIS is available the Amanda system requests GIS lookup the PropertyRSN. The lookup is performed at the lowest level first (unit number). The Amanda system performs multiple lookups to GIS at progressively less granular levels (building, then street address) until a PropertyRSN is returned.
  • If the Amanda system looks up the PropertyRSN, the Amanda system performs the lookup at the lowest level first (unit number). The Amanda system performs multiple lookups at progressively less granular levels (building, then street address) until a PropertyRSN is returned.
  • If the PropertyRSN is not returned the Amanda system assigns the Case to an Inspector based on the Zoning and District located in the message from the Enterprise Service Bus. If an Inspector can not be determined based on the Zoning and District, the Amanda system assigns the Case to the ARR Customer Service team to triage. ARR Customer Service reviews the Case and assigns the appropriate Inspector.

    After the Case is created the Transmit Amanda Data to ESB use case is triggered as the case Status is changed to new.
