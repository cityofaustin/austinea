Create Feedback Request and Triage
Use Case Model

Scope: A Feedback Reporter requests a Feedback Submitter submits Feedback for Austin 311 or the City of Austin on their behalf. The Feedback Submitter creates a ServiceRequest311Feedback which is sent to the 311 Command Center. The 311 Command Center reviews the ServiceRequest311Feedback (identify severity, priority, timeliness, group responsible for resolution). If more information is required to resolve the ServiceRequest311Feedback the 311 Command Center notifies a Feedback Resource (Feedback Report or End User Department). The Feedback Resource provides additional information to the 311 Command Center. The process loops until the 311 Command Center is capable of determining the appropriate Resolution required.

Create Feedback Request and Triage
Activity Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Feedback Submitter/Create Feedback Service Request in CSR System
A Feedback Reporter requests a Feedback Submitter (Austin 311 Agent, CSR system, or CSR user) submits Feedback on their behalf. The Feedback Submitter creates a ServiceRequest311Feedback, answers the pertinent feedback questions and saves the ServiceRequest311Feedback. The ServiceRequest311Feedback sends a Notification to the 311 Command Center indicating a new ServiceRequest311Feedback has been submitted and needs triage.
002 311 Command Center/Review Feedback, Determine Resonse, Update Feedback, and Notify Appropriate Role
The 311 Command Center retrieves the ServiceRequest311Feedback from the CSR system. The 311 Command Center reviews the ServiceRequest311Feedback and determines the response required through the use of a DispositionMatrix. Review includes listening to CallRecordings, researching KnowledgeBase, Websites, contacting End User Departments, Reviewing Emails & Processes. If the ServiceRequest311Feedback has insufficient information to complete triage the 311 Command Center contacts the Feedback Resource (Feedback Reporter or End User Department) and requests additional information on the Feedback. If the ServiceRequest311Feedback was submitted by an internal Austin 311 staff member and involves an Austin 311 Agent transaction, the 311 Command Center coaches the Austin 311 Agent. If the ServiceRequest311Feedback requires a 311 Work Group to resolve the Feedback the 311 Command Center assigns the ServiceRequest311Feedback to the appropriate 311 Work Group (Extension Point). The 311 Command Center answers the Root Cause Flex Note before assigning to appropriate resolution user. Once assigned the CSR system notifies the assigned user of new ServiceRequest311Feedbacks in their queue.
003 Feedback Resource/Provide Additional Information on Feedback
The Feedback Reporter provides additional information on the Feedback to the 311 Command Center.
004 311 Command Center/Coach Staff on Feedback
The 311 Command Center coaches the impacted Austin 311 Agent on the Feedback received. The 311 Command Center updates the assigned ServiceRequest311Feedback with actions taken. CallRecordings are used, as necessary, to coach the Austin 311 Agent.
