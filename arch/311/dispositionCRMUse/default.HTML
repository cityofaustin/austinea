Disposition CRM Use
Use Case Model

Scope: A CRM User requests the system starts a CRMDisposition. Alternatively, the Enterprise Service Bus requests the system starts a CRMDisposition. The system starts the CRMDisposition which creates a new CRMDisposition. For each action the CRM User completes the system updates the CRMDisposition with their action taken (ServiceRequest created, KnowledgeBaseItem accessed, etc.). Additionally, the CRM User provides manual updates to the CRMDisposition to add notes, etc. as necessary. The CRMDisposition is stopped and completed by a CRM User, the system due to a time-out, or from a request from the Enterprise Service Bus. If the system interfaces with a phone system the system creates a COAESBMessage and requests the Enterprise Service Bus records the CRMDisposition in the destination system. If the EnterpriseServiceBusStatus is a non-successful status the system sends Notification to the CRM Manager.

Disposition CRM Use
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Provide Customer Relationship Management/startDisposition
A CRMUser or an interfaced system requests the system starts a CRMDisposition.
002 Provide Customer Relationship Management/updateDisposition
The system updates the CRMDisposition with new DispositionActions as the CRM User completes actions (creating ServiceRequests, searching the CRM, accessing KnowledgeBaseItems, etc.). Additionally, the CRM User updates the DispositionAction with information manually including notes, etc.
003 Provide Customer Relationship Management/stopDisposition
A CRM User stops the CRMDisposition by manually ending the CRMDisposition. Alternatively, the system stops a CRMDispositionbased on a configured schedule or time-out rule managed by the CRM Manager. Additionally, an interfaced system requests a CRMDispositionis stopped based on a trigger in the interfaced system.
004 Provide Customer Relationship Management/createCOAESBMessage
The system creates a COAESBMessage. The system creates a COAESBMessageHeader and adds the InterfaceServiceRequest, CRMDisposition, Resource, etc. to the message body.
005 Enterprise Service Bus/recordDisposition
The system requests the Enterprise Service Bus record the CRMDisposition in the destination system. The Enterprise Service Bus returns a EnterpriseServiceBusStatus.
006 Provide Customer Relationship Management/sendNotification
The system sends a Notification (email, text, etc.) to the recipient. Notification includes Attachments as relevant. If a response from a Citizen is requested the system provides either a URL to respond, generates a subject line for response to the system so the system recognizes the response, etc. The system saves a copy of the Notification after transmission for archival purposes (copy of message, recipients, transmission date/time, schedule, etc.). For public notifications the system sends the Notification per the schedule defined by the Public Information Manager. For ServiceRequest Notifications the system attaches a copy of the Notification with the ServiceRequest for archival purposes. CRM Users resend/cancel Notifications as desired. Notifications are sent ad hoc and also scheduled based on configuration by a CRM Manager. If a delivery receipt is configured the system tracks delivery status of the Notification. For GISLayer updates the system notifies the GISManager identified for the GISLayer. The system includes the number of changes to the GISLayers, date/time of retrieval from GIS, and GISLayer name. For InterfaceServiceRequests, if the system receives an EnterpriseServiceBusStatus back that is not success then the system notifies the CRM Manager. If a KnowledgeBaseItem fails to index the system notifies the KnowledgeBaseItem's Editor with the KnowledgeBaseItem's title and date of failed index.
