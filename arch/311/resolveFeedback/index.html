Resolve Feedback
Use Case Model

Scope: The Feedback Responder receives a ServiceRequest311Feedback. If the Feedback is resolvable by the workgroup the Feedback Responder resolves the Feedback including requesting services from as many 311 work groups as necessary. Sometimes Feedback resolution requires multiple work groups to resolve. If the Feedback is invalid no services are requested to resolve the Feedback. The Feedback Responder contacts the Feedback Submitter to notify them of the Feedback result. The Feedback Responder then updates the ServiceRequest311Feedback with notes on their results and either closes the ServiceRequest, if no other work groups are working the Feedback, or completes their activity on the ServiceRequest while it remains open until all work is completed by each work group.

Activity Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Feedback Responder/Resolve Feedback and Request Services from 311 Work Groups
A Feedback Responder reviews the assigned ServiceRequest311Feedbacks each day and resolves the Feedback as appropriate, or requests the services required to resolve the Feedback (See Use Case diagram for extension points). If staff Feedback, but not 311 Agent Feedback: the Feedback Responder (Chain of Command) coaches staff on the Feedback. If Client Relations & Information Services Feedback: If Feedback concerning external department: The Feedback Responder reviews CalltRecordings, KnowledgeBase, and other research material as needed; primarily to ensure Austin 311 followed process appropriately. The Feedback Responder notifies the External Department of the Feedback through a notification (what's stated by citizen, actions taken, 311 request for action). If Feedback concerning community outreach: Feedback Responder notifies appropriate staff through Notifications (internal website, email) for positive feedback. If Feedback includes a request for attendance at a future Event, The Feedback Responder updates the EventList (NameOfEvent, Month, ContactName, ContactNumber, ContactEmail) with the future Event. If Feedback is negative the Feedback Responder coordinates review with the Chain of Command, and Client Relations & Information Services team, as needed, and updates the appropriate CommunityOutreachPlan (and/or CommunityOutreachMaterials). If Feedback concerning 311 marketing: The Feedback Responder notifies appropriate staff through Notifications (internal website, email) for positive feedback. If Feedback is negative the Feedback Responder coordinates review with the Chain of Command, and Client Relations & Information Services team, as needed, and updates the appropriate MarketingPlan (and/or MarketingMaterials). If Feedback concerning EmergencyResponsePlan: The Feedback Responder notifies appropriate staff through Notifications (internal website, email) for positive feedback. If Feedback is negative the Feedback Responder coordinates review with the Chain of Command, and Client Relations & Information Services team, as needed, and updates the appropriate EmergencyResponsePlan (and/or EmergencyResponseMaterials). If Feedback concerning a client service: the Feedback Responder initiates the Identify Client Need use case. If Feedback concerning a ServiceRequest: the Feedback Responder initiates the Update Service Request Type use case. If Feedback concerning a KnowledgeBaseItem: the Feedback Responder initiates the Update Knowledge Base use case. If Feedback concerning a 311Website: the Feedback Responder initiates the Update 311 Website use case. If System Operations Feedback: If Feedback concerning staff schedule or workforce management: the Feedback Responder starts the Adjust Staff Schedule use case. If Feedback concerning call routing: the Feedback Responder starts the Update Call Routing use case. If Feedback concerning 311 Agent phone skill: the Feedback Responder starts the Modify Agent Phone Skill use case. If Feedback concerning 311 hardware or software (excluding CSR): the Feedback Responder starts the Update 311 Technology use case. If Feedback concerning 311 Agent seating: the Feedback Responder starts the Update Floor Plan Chart use case. If Feedback concerning 311 Agent Scorecard: the Feedback Responder adds the Feedback to the Scorecard for next revision. If Training Feedback: If Feedback concerning TrainingMaterials: the Feedback Responder updates the TrainingMaterial. If Feedback requires staff to be trained: the Feedback Responder starts the Train 311 Staff use case. If Staff Development Feedback: The Feedback Responder starts the Resolve Staff Development Feedback use case.
00X Feedback Responder/Notify Feedback Submitter of Feedback Result
The Feedback Responder notifies the Feedback Submitter of the results of the Feedback (services requested, etc.)
003 Feedback Responder/Update Feedback with Results
The Feedback Responder updates the ServiceRequest311Feedback with notes on the services requested, Notification to the Feedback Submitter, and either closes the ServiceRequest (if no additional response is required), or completes their assigned activity (if additional responses are outstanding).
