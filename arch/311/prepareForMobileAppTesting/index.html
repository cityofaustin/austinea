Prepare for Mobile App Testing
Use Case Model

Scope: Once the MobileAppAccentanceCertificate is received by the Vendor the Chain of Command negotiates the exact MobileAppGoLiveDate. The Chain of Command sends out a go-live Notification to the stakeholders in the mobile app. The Mobile App Lead identifies the tasks required for the mobile app update and then requests services from the required groups. Requested services include service request changes (ServiceRequestTypeChange), knowledge base update (KnowledgeBaseChange), and website update (WebsiteChange). For ServiceRequestTypeChanges the Initiate Service Request Change use case is invoked. For KnowledgeBaseChanges the Update Knowledge Base use case is invoked. For WebsiteChanges the Update 311 Website use case is invoked. While the requested services are being completed per their respective use cases the Mobile App Lead updates the MobileAppTestPlan. The Business Process Specialist notifies the Client Departments of the go-live date (MobileAppGoLiveDate). The Vendor completes the mobile application update either by updating the 311MobileApplication with enhancements, or adding/removing/modifying MobileAppServiceRequestTypes. After all requested services are completed, MobileAppTestPlan updated, Client Departments notified, and the mobile application updated the mobile application is ready for testing. If a production go-live is pending the Chain of Command sends a final go-live Notification 24-hours in advance.

Prepare for Mobile App Testing
Activity Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Chain of Command/Identify Go-Live Date for Changes with Vendor and Notify Stakeholders
The Chain of Command identifies a go-live date (MobileAppGoLiveDate) with the Vendor based on the complexity of the MobileAppChangeDefinition and resource availability. MobileAppGoLiveDate includes Presentation and Production. The Chain of Command notifies Stakeholders of the MobileAppGoLiveDate.
002 Mobile App Lead/Identify Tasks for Mobile Application Update and Request Services From Groups
The Mobile App Lead is notified of the MobileAppGoLiveDate and identifies the deliverables and tasks required to prepare for the Mobile App Update based on the completed MobileAppDefinition. The Mobile App Lead requests services from the appropriate work groups (ServiceRequestChange, WebsiteRequest, KnowledgeBaseChange, etc.).
003 Mobile App Lead/Update Test Plans
The Mobile App Lead updates tests plans, and creates new test plans as necessary, to fully test the mobile app changes per the MobileAppDefinition.
004 Initiate Service Request Change

005 Update Knowledge Base

006 Update 311 Website

007 Business Process Specialist/Notify Clients of Go-Live Date for Changes
The Business Process Specialist notifies the impacted Client Departments of the Go-Live date for Mobile App Changes (email and CalendarInvite required).
008 Vendor/Configure 311 Mobile Application
The Vendor configures the 311MobileApplication with new features, new MobileAppServiceRequestTypes, or updates MobileAppServiceRequestTypes. The Vendor sends a Notification when the changes have been completed in the 311MobileApplication.
009 Chain of Command/Send Notification of Go-Live 24-Hours in Advance
The Chain of Command sends a go-live notification for the production launch of the mobile application changes.
