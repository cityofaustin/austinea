Lookup GIS Information
Use Case Model

Scope: The system looks up a Location based on the DisplayAddress provided (also includes intersections, map point selections, common locations, etc.). If an external GIS system is available the system performs the Location lookup in the external GIS system. If an exact match for the DisplayAddress is not identified the system suggests nearest matches for the Location to the CRM User. The CRM User selects the Location from the available suggestions. If the Location provided is for a ServiceRequest the system reviews the Location, ServiceRequestType, and determines the appropriate response. The system then performs one of several actions based on the provided Location: If the Location is invalid the system notifies the CRM User of the invalid Location. If the Location is outside the jurisdiction for the ServiceRequest the system notifies the CRM User. If the Location is valid the system performs a GIS data lookup. Alternatively, the system performs the GIS lookup with an external GIS system. If the CRM User requests additional GIS data for the Location the system displays the relevant information. If LocationAlerts are identified for the Location the system notifies the CRM User of the LocationAlert. If the Location and ServiceRequest identify the ServiceRequest as a duplicate the system updates the ServiceRequest as a duplicate.

Lookup GIS Information
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Provide Customer Relationship Management/lookupLocation
A CRM User provides an DisplayAddress to the system (street address, intersection, map point, building name, etc.). The system looks up the Location based on the DisplayAddress and identifies the nearest closest Location (address point, street address) in the City's Addressing.
002 GIS/lookupLocation
The system requests a Location lookup from a GIS system (City of Austin, ArcGISOnline, Google, Apple, etc.). The GIS system returns the nearest valid Location based on the address provided.
003 Provide Customer Relationship Management/suggestLocation
The system provides a list of suggested Locations to the CRM User based on percentage match with the provided DisplayAddress. The percentage is managed by the GIS Manager.
004 Provide Customer Relationship Management/selectLocation
The CRM User uses the system and selects a Location from the suggested Locations.
005 Provide Customer Relationship Management/reviewLocationForServiceRequest
The system reviews the provided Location and the ServiceRequestType's LocationSettings to determine if the Location is invalid, outside of jurisdiction, or valid.
006 Provide Customer Relationship Management/notifyCRMUserOfInvalidLocation
The system notifies the CRM User of an invalid Location. Notification message is customizable by GIS Manager.
007 Provide Customer Relationship Management/notifyCRMUserOfOutOfJurisdictionLocation
The system notifies the CRM User the Location is out of jurisdiction. Notification message is customizable by GIS Manager.
008 Provide Customer Relationship Management/lookupGISData
The system looks up relevant GISLayers for the Location.
009 GIS/lookupGISData
The system requests GIS looks up GISLayers for the Location.
010 Provide Customer Relationship Management/lookupGISDrillDownData
The CRM User requests the system looks up additional drill down GISLayer information for the Location. The CRM User provides a CRMSearch to the system to identify the specific drill down GIS information (GISLayers, GISMaps, etc.)
011 Provide Customer Relationship Management/displayLocatoinAlert
The system displays a LocationAlert, a customized message configured by a Service Request Manager, GIS Manager, to the CRM User associated to the Location.
012 Provide Customer Relationship Management/updateServiceRequestAsDuplicate
The system notifies the CRM User of duplicate ServiceRequests found. Duplicate check includes: proximity, date range, street address [building, suite, unit], status, question/answers, activities/outcomes based on ServiceRequestType configuration by Service Request Manager. The system links the two ServiceRequests and the second ServiceRequest is modified: removal of Activities, resolution specific Questions, updated Status indicating duplicate Status. When the original ServiceRequest is completed the duplicate ServiceRequest(s) are closed automatically by the system. CRM Users have the option to break the duplicate link to treat the new ServiceRequest as a unique instance of an issue.
