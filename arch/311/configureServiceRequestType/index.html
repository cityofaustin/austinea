Configure Service Request Type
Use Case Model

Scope: A CRM User accesses the system and requests the system creates a ServiceRequestChange. The CRM User identifies ServiceRequestType change details, along with the Service Request Manager completing the change, and the system updates the Service Request Manager's WorkQueue. The Service Request Manager queries their WorkQueue for the new ServiceRequestChange and uses the system to complete the changes to the ServiceRequestType. The Service Request Manager uses the system to promote the change to the desired CRM system (typically to Development, Training and then Production later). Alternatively, the Service Request Manager requests the system completes a bulk creation/update to multiple ServiceRequestTypes. The system updates each ServiceRequestType and publishes the changes to the desired system. After completing the change the Service Request Manager updates the ServiceRequestChange and the system updates the WorkQueue for the Chain of Command (often the same as the CRM User that submitted the change request). The Chain of Command uses the system to access their WorkQueue and then reviews the change to determine if it's correct. The Chain of Command updates the ServiceRequestChange with their notes and approval/rejection. The system updates the WorkQueue for the Service Request Manager. If rejected, the Service Request Manager updates the ServiceRequestType and promotes to the same CRM system again for review. If approved, the Service Request Manager publishes the ServiceRequestType changes to other CRM systems. The review process loops until the ServiceRequestType is updated in the final requested CRM system (typically production). If the ServiceRequestChange requires acknowledgment the system updates the WorkQueues for the requested CRM Users and the Acknowledge CRM Change use case is invoked.

Configure Service Request Type
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Provide Customer Relationship Management/createServiceRequestChange
A CRM User accesses the system and creates a ServiceRequestChange. The CRM User provides ServiceRequestType, description of changes, reason for change, Service Request Manager to complete the work, Point of Contact. The system updates the WorkQueue for the identified Service Request Manager.
002 Provide Customer Relationship Management/updateWorkQueue
The system updates the CRM User's WorkQueue to include the KnowledgeBaseItem's requirement for acknowledgment. CRM User accesses the system and updates the WorkQueue to identify acknowledgment of KnowledgeBaseItem. After completed the CRM User's WorkQueue removes the KnowledgeBaseItem from the WorkQueue. The system updates the CRM User's WorkQueue indicating a KnowledgeBaseItem did not index properly, and requires CRM User review. The system updates the Editor's (CRM User) WorkQueue indicating a KnowledgeBaseItem received a Comment and requires CRM User review.
003 Provide Customer Relationship Management/queryWorkQueue
A CRM User requests the system queries a WorkQueue. The CRM User provides a variety of information to identify a WorkQueue, or subset of WorkQueue including: ServiceRequest information (CRMSearch, i.e. Status, Type, Date/Time, Outcome, Activity, Question/Answers, Location, GIS Information, Service Level Agreement,), and any ServiceRequest information is available to use for query. Queries are sortable by CRM User for any criteria provided.
004 Provide Customer Relationship Management/createUpdateServiceRequestType
The Service Request Manager accesses the system and creates/updates the ServiceRequestType per the instructions in the ServiceRequestTypeChange. The system allows Service Request Managers to configure questions/answers, activities/outcomes, resource assignments, programming to control question/activity/assignments based on triggers (if/then/case statements), notifications, interfaces, etc. ServiceRequestType changes aren't visible to CRM users until published by a Service Request Manager to the desired CRM system.
005 Provide Customer Relationship Management/publishServiceRequestType
The Service Request Manager requests the system publishes a ServiceRequestType to the desired CRM system.
006 Provide Customer Relationship Management/createUpdateBulkServiceRequestType
The Service Request Manager requests the system perform a bulk creation/update to multiple ServiceRequestTypes. The Service Request Manager provides an Attachment with the requested changes.
007 Provide Customer Relationship Management/updateServiceRequestChange
A CRM User accesses the system and updates the ServiceRequestTypeChange with notes on their work progress. CRM Users track their progress through a work percentage attribute, and requests the system add the ServiceRequestChange to the WorkQueue of the desired CRM User based on the progress of the ServiceRequestTypeChange.
008 Chain of Command/reviewServiceRequestTypeChange
The Chain of Command, most often a Business Process Specialist, reviews the ServiceRequestType and the associated changes made by the Service Request Manager to determine if the changes satisfy the ServiceRequestTypeChange. If the changes are rejected the Chain of Command updates the ServiceRequestTypeChange with notes on revisions required. If changes aren't required the Chain of Command updates the ServiceRequestTypeChange with approval notes.
