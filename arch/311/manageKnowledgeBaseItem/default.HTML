Manage Knowledge Base Item
Use Case Model

Scope: A Knowledge Base Manager requests the system to create/update a KnowledgeBaseItem. Alternatively, if many KnowledgeBaseItems need to be created or updated, the Knowledge Base Manager uploads an Attachment to the system for a bulk creation/update by the system. The system performs the KnowledgeBaseItem creation/update for each item in the Attachment. The KnowledgeBaseItem is published to the desired CRM system either manually by the Knowledge Base Manager or automatically by the system depending on the disposition of the Attachment file. If an index is required to make the KnowledgeBaseItem searchable, the KnowledgeBaseItem is either indexed manually by the Knowledge Base Manager, or the system indexes the KnowledgeBaseItem per an IndexSchedule configured by the Knowledge Base Manager. If the index fails, the system notifies the KnowledgeBaseItem's Editor. If public sharing is selected for the KnowledgeBaseItem the system requests either a Website or Social Media to post an update on the KnowledgeBaseItem. If the KnowledgeBaseItem requires acknowledgment the system updates the CRM Users' WorkQueues. The Acknowledge CRM Change use case is invoked.

Manage Knowledge Base Item
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Provide Customer Relationship Management/createUpdateKnowledgeBaseItem
The Knowledge Base Manager, or the system, accesses the system and requests the system creates a new KnowledgeBaseItem, or update an existing KnowledgeBaseItem. The Knowledge Base Manager provides (owning group, start date, stop date, title, name, description, keywords, file, associated ServiceRequests, associated files/URLs/Searches/FAQs, version, isAcknowledged per version, users required to acknowledge, change list, user comments, search depth, KB maintainer/responsible for editing/updating, interval for review, next review date, view count, isPublished, private information, public information). The Knowledge Base Manager must select the isPublished option in the system for the KnowledgeBaseItem change to appear. The system maintains old versions of KnowledgeBaseItems for historical review and auditing purposes. If a Comment is received the system attaches the Comment to the KnowledgeBaseItem.
002 Provide Customer Relationship Management/publishKnowledgeBaseItem
A Knowledge Base Manager requests the system publishes the KnowledgeBaseItem to the desired CRM system. Alternatively, if the system performs a bulk KnowledgeBaseItem creation/update the system publishes the KnowledgeBaseItem to the desired CRM system per the Attachment's disposition.
003 Provide Customer Relationship Management/createUpdateBulkKnowledgeBaseItems
The Knowledge Base Manager accesses the system and uploads an Attachment containing the bulk KnowledgeBaseItem update, including creation or modification.
004 Provide Customer Relationship Management/indexKnowledgeBaseItem
The system indexes the KnowledgeBaseItems on a scheduled interval (new, changed KnowledgeBaseItems based on group scope, interval [time of day, day of week, time elapsed, trigger by status or change to KnowledgeBaseItem]), controlled by the Knowledge Base Manager, or ad hoc per Knowledge Base Manager request. If the index encounters an error the system generates an update to the KnowledgeBaseItem's Manager for review (updateUserWorkQueue, information on error: reason, time). After indexing the KnowledgeBaseItem is available for searching by CRM Users, the public, etc.
005 Provide Customer Relationship Management/sendNotification
The system sends a Notification (email, text, etc.) to the recipient. Notification includes Attachments as relevant. If a response from a Citizen is requested the system provides either a URL to respond, generates a subject line for response to the system so the system recognizes the response, etc. The system saves a copy of the Notification after transmission for archival purposes (copy of message, recipients, transmission date/time, schedule, etc.). For public notifications the system sends the Notification per the schedule defined by the Public Information Manager. For ServiceRequest Notifications the system attaches a copy of the Notification with the ServiceRequest for archival purposes. CRM Users resend/cancel Notifications as desired. Notifications are sent ad hoc and also scheduled based on configuration by a CRM Manager. If a delivery receipt is configured the system tracks delivery status of the Notification. For GISLayer updates the system notifies the GISManager identified for the GISLayer. The system includes the number of changes to the GISLayers, date/time of retrieval from GIS, and GISLayer name. For InterfaceServiceRequests, if the system receives an EnterpriseServiceBusStatus back that is not success then the system notifies the CRM Manager. If a KnowledgeBaseItem fails to index the system notifies the KnowledgeBaseItem's Editor with the KnowledgeBaseItem's title and date of failed index.
006 Website/postUpdate
The system requests a Website posts an update containing the message contents (text, etc.). The message contains an expiration date for the Website post.
007 Social Media/postUpdate
The system requests the Social Media platform posts an update on the requested SocialMediaSMSServiceRequest and disposition (created, cancelled, etc.), or public notification (text).
008 Provide Customer Relationship Management/updateWorkQueue
The system updates the CRM User's WorkQueue to include the KnowledgeBaseItem's requirement for acknowledgment. CRM User accesses the system and updates the WorkQueue to identify acknowledgment of KnowledgeBaseItem. After completed the CRM User's WorkQueue removes the KnowledgeBaseItem from the WorkQueue. The system updates the CRM User's WorkQueue indicating a KnowledgeBaseItem did not index properly, and requires CRM User review. The system updates the Editor's (CRM User) WorkQueue indicating a KnowledgeBaseItem received a Comment and requires CRM User review.
009 Provide Customer Relationship Management/submitCommentOnCRM
CRM User requests the system to submit CRMComment and the system prompts the CRM User for their comment. Comment is added to the appropriate CRM system item (KnowledgeBaseItem, GISLayer, etc.) and the WorkQueue updated for the appropriate CRM User.
