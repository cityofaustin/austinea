Add PWA Licenses
Use Case Model

Scope: The Customer Contacts the HelpDesk and requests a PWA license. The HelpDesk generates a request and assigns it to the Project Online Administrator. The Project Online Administrator contacts the PMO Manager for approval. The PMO Manager contacts the customer to verify the needs (this determines the level of customer license request required for the customer.) The PMO manager contacts the Project Online Admin and requests they continue the licensing process. The Project Online Admin creates a work order for the Buyer within the helpdesk. The Buyer first contacts the vendor and gets a quote for the software. The Vendor provides a quote, which the Buyer forwards to the customer, and requests the customer's funding code information. The Customer provides funding codes, and a list of users to which the licenses will be assigned, and requests the Buyer create a purchase order. The Buyer uses the helpdesk software to create a purchase order. The vendor contacts the Buyer to indicate sale completion. The Buyer contacts the Office 365 Admin, and the PMO Online Admin indicating licenses are available for assignment. The Office 365 Admin assigns the specified users to the appropriate lists for access and notifies the Project Online Admin of completion. The Project Online Admin uses the list and adds the users to the Project Online user rosters. Depending on the level of the license, Enterprise Project Types (EPTs) are created within Project Admin (this is done in Pilot environment). At this time the Project Online Admin has the customer sign the Service Level Agreement ( and the customer provides any logo branding they desire). If branding is provided, the Project Online Admin applies the desired branding and then completes the process by creating breakdown structures for Pilot, Test, and Production Environments and closing the helpdesk ticket.
