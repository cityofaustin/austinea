Move File To FTP Site
Use Case Model

Scope: The Customer uses the Operating system to copy a file from their disk location into the designated FTP site. The operating system displays an acceptable use banner, and prompts the user to acknowledge and confirm their data is in accordance with City policy related to placing data on an FTP site. Once confirmed, the Operating system moves the file to the FTP location. Once the copy to the new location is complete, DLP services evaluate the file and determine if there is a pattern match indicating potential policy violation. If issues are perceived, the DLP software moves the offending file into a "Quarantine" area for further evaluation. The DLP software sends email to a distribution list of Security team members, as well as notifies the Customer of a potential problem. Security evaluates the quarantined file. If problems are confirmed, Security notifies the Chain of Command (of the Customer) of the issue, and determines next steps. If a false positive was reported, the Security Team moves the quarantined file into FTP site, and notifies the Customer of file acceptance.

Move File To FTP Site
Sequence Diagram

Req# Type/Interface - Requirement Description Performance
001 Operating System/copyFileToFTPSite
The Customer uses the Operating System to move a file to their FTP location.
002 Operating System/displayWarningBanner
The Operating System detects a new file has been placed into an FTP Directory and displays an "acceptable use policy", and requests the customer confirm understanding.

003 Customer/confirmFTPRuleUnderstanding
The Customer confirms their understanding of the "Acceptable Use Policy" by checking a box, or clicking a visual queue on their screen. NOTE: If the Customer does not provide this confirmation, the operating system does not copy the requested file to the FTP location.
004 Operating System/moveFileToFTPSite
Having received a confirmation of understanding of the "Acceptable Use Policy" for FTP sites, the operating system copies the designated file into the FTP directory as requested by the Customer.
005 DLP Services/detectNewFileAndRunDLPChecks
The DLP Service monitors the FTP directory, and upon successful movement of a new file into an FTP directory, runs a scan of the file placed there - checking it for various policy violations ( credit card numbers, social security numbers, citizen private data, etc.).
006 Operating System/moveFileToQuarantine
Having detected a violation of Acceptable Use Policy, the DLP Service uses the Operating system to move the file from the FTP Site to a quarantine area.
007 DLP Services/notifyCustomerOfIssue
The DLP Service notifies the customer via email of issues with their file, and indicates research will be conducted on their file before allowing it to be released to their FTP site.
008 DLP Services/notifySecurityOfIssue
The DLP Service notifies the Security Team (a configurable distribution list) via email with the Customer's filename, information about errors perceived, Customer name and contact information, Customer Supervisor name, FTP Site Location, File name, Quarantine Location and where the file now resides.
009 Security/evaluateFile
The Security team evaluates the File and determines if, in fact, a policy violation has occurred with the file, as indicated by the DLP Service.
010 DLP Services/releaseFileForDistribution
The Security team uses DLP Services to indicate the pattern located within the Customer's file is acceptable, preventing false positives from occurring for this specific file again.
011 Operating System/moveFileFromQuarantineToFTPSite
The Security Team uses the Operating system to move the file from the Quarantine area back to the Customer FTP Site.
012 Security/notifyCustomerFileClearedForDistribution
The Security Team sends email to the Customer confirming successful clearance of any perceived issues with the file distribution, and communicating movement of the file from Quarantine to the Customers FTP Site.
013 Security/notifyChainOfCommandAboutFileIssue
The Security Team sends email to the Chain of Command, confirming Acceptable Use Policy Violations with the file distribution, and suggesting next steps.
