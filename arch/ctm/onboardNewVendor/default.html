Add Project to Project Web Access
Use Case Model

Scope: The Project Manager determines a need for Vendor assistance on a project and initiates the network and system access. They submit a helpdesk ticket to begin the process. The Helpdesk routes the request for Vendor Access to the Security team who contacts the Vendor and sends the Vendor Access forms. The Vendor completes the access forms and returns them to Security. Security Performs a background check on the vendor candidate. If access is denied, Security Contacts the Project Manager and indicates Denial of access. The Project manager then contacts the vendor to indicate the failure and closes the helpdesk ticket. If access is accepted, Security contacts the vendor and the project manager to indicate the approval and closes the helpdesk ticket. The project manager contact the Project Online Admin to add the vendor Resource. The Project Online Admin submits a helpdesk ticket to add the vendor candidate to active directory for whatever resources are necessary for the project. HelpDesk routes the request to the Office 365 Administrator, who configures the necessary active directory access for the vendor and closes the Helpdesk Ticket. The Project Online Admin adds the resource to the Account Pool based on the approved access level. If a new Project Web Online license is required to accommodate the new resource, the Project Online Admin performs the "ADD PWA LICENSES" process.
