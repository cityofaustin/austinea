Add Project to Project Web Access
Use Case Model

Scope: The Project Submitter (either a Project Manager or the Chain of Command) fills out a project creation form and provides it to the Portfolio Manager. The Portfolio Manager evaluates the form, and works with the submitter to correct or complete any missing information. Once acceptable, the Portfolio Manager adds the project to the on Premise Portfolio Sharepoint List (within 72 business hours), and sends email to the Project Online Admin to create the project in Project Web Access. The Project Online Admin creates the project in Project Web Access within 48 business hours and sends email to the Portfolio Manager confirming project creation. The Portfolio Manager enters the remaining information into Project Web Access within 24 business hours, sending notification back to the Project Online Admin when completed. Lastly, the Project Online Admin adds the project name to the site collection column in project Web Access and sends email to the Project Submitter confirming completion.
