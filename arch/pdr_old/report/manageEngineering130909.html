Use Case
Obtain Final Inspection

Use Case Scope: A Permit Applicant uses a Web interface to submit an Inspection Request (and/or supporting inspection documentation). When entered properly and quality checked by the system, the system adds the Inspection Request to an Inspection Queue and provides the Permit Applicant details relevant to the request such as anticipated inspection timing, city point of contact (POC), etc. The Inspector Manager may review the Inspection Queue, override system automated selections by reassigning inspector assignments or make adjustments to the inspection information. When changes are made to the Inspection Queue relevant to the Permit Applicant, a notification is sent to the Permit Applicant informing them on changes and/or updates to the Inspection Request. The system provides the Inspector a geographic information system (GIS) map with priority ordered (overlay) indication of inspection locations. The Inspector may use the map to navigate to the inspection location or retrieve relevant information regarding the inspection. Once inspected, the system provides the Inspector the ability to record their inspection results, which automatically updates the Inspection Queue. If the Inspection Request is a building final inspection, the system provides the ability for the Permit Applicant to retrieve and print Certificate of Occupancy or Certificate of Compliance.

Table of Interface Descriptions
Obtain Final Inspection

Type/InterfaceDescription
Issue Permits/
requestInspection		 The system provides the Permit Applicant the ability to review Project Information with a list of permits issued and select the desired permit issuance for inspection. Alternately, the system provides a method to request inspection for single purpose permits for selected trade contractors performing routine day-to-day activities requiring permits. For selected permits requiring additional documentation, the system provides a method for the Permit Applicant to browse and upload additional documentation requirements associated with the issued permit - this may be a result of a previous inspection identified by the Inspector. The system assigns an Inspector and provides appropriate contact information for inspection timing and coordination (Inspection Queue). Inspections typically occur 24 - 48 hours from the request. The Permit Applicant can also see administrative hold on a permit - i.e., revision required, inspection resulted in work out of scope of the permit. etc. Finally, the system provides for special approved program inspections that provide a means for the Permit Applicant to schedule an inspection for a specific date and time.
Issue Permits/
updateInspectionQueue		 Once an Inspection Request is submitted, the system updates the Inspection Queue by assigning an Inspector based on the area of responsibility. The system updates the Inspection Request Geographic Information Systems (GIS) database with pertinent information related to the Inspection Request (ex., design documents from intake Submittal Package, relevant Project Information, location, site access information, point of contact information, etc.). For special inspections based on city rules for selected registered contractors performing routine single purpose trade permits, the system randomly adds one inspection for every five single purpose permits requested.
Issue Permits/
reassignInspectionInformation		 The system provides the ability for an Inspection Manager to modify the Inspection Queue to include Inspector assignment, priority order, and inspection timing. The Inspection Manager may also perform a quality review of Inspection Requests.
Permit Applicant/
reviewInspectionNotification		 As updates occur in the Inspection Queue, the system calculates new anticipated inspection times and automatically notifies the Permit Applicant on the new anticipated inspection time. Message also includes inspection results.
Issue Permits/
identifyInspectionItem		 The system provides the Inspector a geographic information system (GIS) map with priority ordered (overlay) indication of inspection locations. The Inspector must be able to quickly and easily modify the priority order based on proximity location to inspection location and ability to annotate why priority order was modified. When an inspection location icon is selected, the Inspector selects location navigation to navigate to the GPS location or to obtain additional inspection information such as permit Inspection Request, relevant design documentation from intake of the Submittal Package, code and compliance details, coordination details (i.e., codes to access site), etc. The Inspector must be able to override and reassign individual inspections and inspection assignments.
Issue Permits/
recordInspectionResults		 The system provides the ability for the Inspector to record inspection results. The system provides an Inspector modifiable pick-list of inspection results for routine inspections. The selected pick-list items must be able to be edited for relevant inspection results.
Issue Permits/
retrieveCertificate		 Once a final building inspection is approved, the Review Inspection Notification message provides a link that the Permit Applicant uses to retrieve and print the Certificate of Occupancy or Certificate of Compliance depending on the inspection type.
