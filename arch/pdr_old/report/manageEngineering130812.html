This report contains the details of our last meeting regarding the Unified Modeling Language (UML) Use Case to Obtain Permits. This is a continuance from our previous Use Case to Perform Regulatory Review / Review Submittal Package. You can follow the link to this report. As an update to our previous work, we added two new messages (interfaces) to the architecture to accommodate the creation of a Project Folder as an interface to the Issue Permits capability (i.e., AMANDA). Throughout the architecture we used Issue Permits as the capability verses the actual system AMANDA. This is standard architecture practice as not to impose a solution onto the system; rather, explore the capability in it's ideal form. The following diagram (last two messages) and table provides just the update to our previous Use Case model.

Review Submittal Packages

Type/InterfaceDescription
Perform Design Review/
createProjectInformation		 Once a Submittal Package obtains final approval, the system generates the necessary Project Information compatible with the Issue Permits capability. The Project Information intakes all relevant attributes of the Submittal Package to include relevant diagrams, project information, contact information, personal identification number (PIN), contractor, identified permit requirements, etc. to automatically populate the Issue Permits system.
Issue Permits/
createProjectFolder		 Project Information is used to create a Project Folder (case package) containing all the relevant information required to process and obtain city permits.

Sequence Diagram Obtain Permits

Table of Interface Descriptions
Obtain Permits

Type/InterfaceDescription
Issue Permits/
obtainPermit		 During the final review and approval of a Submittal Package, the review Notification provides a link (URL) with Project Information encoded in the link containing information that directs the Submittal Applicant to the correct location to obtain required permits. This message directs the Submittal Applicant to the proper URL where the system looks up relevant permit information.
Issue Permits/
lookUpPermitStatus		 The system looks up the Project Information encoded into the obtainPermit URL and presents the Submittal Applicant with options to submit for selected permits previously identified during the Perform Regulatory Review process. The Submittal Applicant owner (owner is defined as residing one calendar year with homestead exemption) is also provided options to transfer building permit and cancel trade permit (if permit was previously issued).
Issue Permits/
selectPermitOption		 The Submittal Applicant selects the desired permit for the system to issue.
Issue Permits/
determineRemittance		 Based on the permit selected, the system automatically determines the Remittance required and presents the Remittance information to the Submittal Applicant with indication of selected form of payment (i.e., credit card, ACH, etc.). The Remittance amounts must be modifiable by identified city staff and not hard coded into the system.
Issue Permits/
payRemittance		 Using traditional online Electronic Financial Services approved by the city, the Submittal Applicant pays the required Remittance and receives Acknowledgment of payment.
Issue Permits/
issuePermit		 When the system receives the proper Acknowledgment that the payment transaction was successful, the system prepares the selected permit in a printable form (i.e., PDF, etc.).
Issue Permits/
acknowledgeInspectionResponsibility		 Once a permit is prepared by the system, the system indicates to the Submittal Applicant responsibility for proper inspection and provides selected indication that this responsibility is accepted before a printable permit is issued by the system.
Submittal Applicant/
printPermitAsRequired		 Once the Submittal Applicant acknowledges inspection responsibility, the system provides a printable permit to display on construction site if required.
Issue Permits/
transferBuildingPermit		 There are circumstances when the permit owner (Submittal Applicant) may wish to transfer a building permit to a prescribed contractor. The Submittal Applicant enters relevant information into the system that clearly identifies the contractor receiving the transferred building permit and stores it into the system. Once transferred, the contractor plays the role of Submittal Applicant to obtain future permits relevant to the Project Information associated with the case file building permit.
Issue Permits/
cancelTradePermit		 The Submittal Applicant (owner) can cancel trade permits already issued. Owner is defined as residing one calendar year with homestead exemption. If an inspection already occurred at the time of cancellation, there is no refund of Remittance; otherwise, a percentage of the Remittance is returned to Electronic Financial Service account. If a refund is issued, a new contractor must pay 100% of the required Remittance. If no refund issued, the contractor must pay 50% of the required Remittance to reissue the permit.
