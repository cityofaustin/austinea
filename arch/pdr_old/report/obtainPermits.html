Use Case Specification for Obtain Permits

Select for larger view

Use Case Diagram Obtain Permits

Use Case Scope: Permit Applicant obtains single purpose trade permits or manages permits relevant to a Submittal Package/Project Information. The system provides Web interface application forms relevant to the permit requested, calculates required remittance and adds permit application to a Review Queue for Reviewer review and comment when required. Permit Applicants may also transfer Building Permits to prescribed contractors, cancel Trade Permits, and/or reactivate Expired Permits.

Table of Interface Descriptions

Category/InterfaceDescription
Issue Permit/
setupRequiredPermits		 Although permits were initially identified and setup by Perform Regulatory Review, the system provides the ability for Submittal Coordinators to review and determine additional permit requirements or required changes. The system provides the ability for the Submittal Coordinator to adjust permit requirements for any Submittal Package. The system provides the ability to administer various states of permits such as overriding normal day-to-day transactions such as suspension or revocation of permit.
Authorized Permit Person/
reviewNotification		 The system provides Notification via text message or email when permits are ready for issuance. Notification may also include other indications of permit status.
Issue Permit/
associateSubmittalPackages		 The system provides the ability to associate Submittal Packages. For example, a building permit application may require completion of a land use re-zoning case.
Issue Permit/
issueSinglePurposePermit		 The Authorized Permit Person selects the desired trade permit (or mobile home building permit) for the system to issue. This function is limited to only the alternate flow to [request a permit] or [reactivate expired permit] and designated for selected trade contractors performing routine day-to-day activities requiring permits. When a mobile home is involved (only in mobile home parks), this requires issuance of a building permit to associate relevant trade permits to the project. This type of building permit can be issued as a single purpose permit.
Issue Permit/
obtainPermit		 During the final review and approval of a Submittal Package, the Review Notification provides an encoded link (URL) that directs the Authorized Permit Person to the associated Project Information and location to obtain required permits - the system provides a list of available permit requests. When a particular permit is selected, the link directs the Authorized Permit Person to the proper URL where the system looks up relevant permit information, status and provides various permit options.
Issue Permit/
lookUpPermitStatus		 The system looks up the Project Information encoded from the obtainPermit URL and presents the Authorized Permit Person with options to submit for selected permits previously identified during the Perform Regulatory Review process and/or updated by the Submittal Coordinator. The Authorized Permit Person is provided options based on available permits and their status.
Issue Permit/
selectPermitOption		 The Permit Applicant selects from the permitting options available depending on the Permit Status of the project. The applicant may request a permit, transfer a building permit, cancel a trade permit, reactivate expired permit, request extension, and withdraw permit.
Issue Permit/
determinePermitApplication		 Based on the Permit Applicant selection to request permit, the system provides the appropriate permit application form for the Permit Applicant to compete and submit. Permit Applicant attribute values known by the system automatically populate relevant values on the form.
Issue Permit/
completePermitApplication		 The Authorized Permit Person uses the system to complete the permit application. Known values of the system automatically populate relevant attributes of the form.
Issue Permit/
determineRemittance		 Based on the permit selected, the system automatically determines the Remittance required and presents the Remittance information to the Submittal Applicant with indication of selected form of payment (i.e., credit card, ACH, etc.). The Remittance amounts must be modifiable by identified city staff and not hard coded into the system.
Issue Permit/
addToRemittanceQueue		 The Authorized Permit Person uses the system to process multiple permit requests (if needed). The system adds remittance to a Remittance Queue until completing permit requests.
Issue Permit/
payRemittance		 Using traditional online Electronic Financial Services approved by the city, the Authorized Permit Person pays the required Remittance and receives Acknowledgment of payment.
Electronic Financial Service/
processRemittance		 The system provides an interface to city approved financial vendor to process online electronic transactions.
Issue Permit/
addToReviewQueue		 The system provides a modifiable rules engine that places certain permit requests into a review queue for Reviewer to ensure completeness and relevance to the request. Common trade permits by established contractors may not require review, but the rules engine may require some trade permit requests to be reviewed.
Issue Permit/
reviewPermitApplication		 The Reviewer selects the Permit Application from the Review Queue and reviews for completeness and accuracy. If approved, the Permit Applicant is provided a URL encoded link to select to acknowledge inspection responsibility and permit issuance. If declined, the Notification provides indication of the deficiencies of the Permit Application and provides a URL encoded with information to redirect the Permit Applicant to the application to correct and resubmit.
Authorized Permit Person/
reviewNotification		 The system provides Notification via text message or email when permits are ready for issuance. Notification may also include other indications of permit status.
Issue Permit/
acknowledgeInspectionResponsibility		 Once a permit is prepared by the system, the system indicates to the Permit Applicant responsibility for proper inspection and provides selected indication that this responsibility is accepted in the form of an Acknowledgment before a printable permit is issued by the system.
Issue Permit/
issuePermit		 When the system receives the proper Acknowledgment of inspection responsibility, the system prepares the selected permit in a printable form (i.e., PDF, etc.).
Issue Permit/
transferBuildingPermit		 The system provides an Authorized Permit Applicant the ability to transfer a building permit to a prescribed contractor. The Authorize Permit Applicant enters relevant information into the system that clearly identifies the contractor receiving the transferred building permit and stores it into the system. Once transferred, the contractor plays the role of Authorized Permit Applicant to obtain future permits relevant to the Project Information associated with the case file building permit. The system provides a means for supporting letters and/or documents to uploaded to the system.
Issue Permit/
reviewPermitRequest		 The permit Reviewer reviews the permit request to transfer building permit or request a permit extension along with any supporting documentation such as contractor Transfer Letter and authorizes or denies permit request.
Issue Permit/
cancelTradePermit		 The Authorized Permit Person uses the system to cancel trade permits already issued. If an inspection already occurred at the time of cancellation, there is no refund of Remittance; otherwise, a percentage of the Remittance is returned to Electronic Financial Service account. If a refund is issued, a new contractor must pay 100% of the required Remittance. If no refund issued, the contractor must pay 50% of the required Remittance to reissue the permit.
Issue Permit/
determineExpiredPermits		 The system determines the relevant expired permit(s) associated with the Submittal Package Project Information and presents to the Authorized Permit Person for selection to reactivate.
Issue Permit/
reactivateExpiredPermit		 The Permit Applicant uses the system to reactivate an expired permit. This requires a form to enter pertinent information (such as reason for reactivation) prior to reactivation.
Issue Permit/
determinePermitFee		 Based on the selected permit requested to be reactivated, the system determines the necessary permit fee.
Issue Permit/
requestPermitExtension		 The Authorized Permit Person uses the system to request a one time extension by completing an online form. The system provides the ability to upload any supporting documentation.
Issue Permit/
withdrawPermit		 The Authorized Permit Person may withdraw a permit possibly due to the fact that they are no longer performing the work - this requires a complete resubmission for a permit.
