Manage Public Information Requests
Use Case Model

Scope: A Citizen contacts the Public Information Office (PIO) and requests information from/about Animal Services. The PIO enters the request into the IQ System, and contacts the Admin Personnel to let them know the information requested. The Admin Personnel runs reports and checks resources to determine what's required to fulfill the request. The labor requirements for the provisioning of the information is determined by the Admin Personnel and entered into the IQ System. The Admin Personnel then contacts the Citizen to determine if they are willing to pay whatever fees are necessary for the data production. If the Citizen wants to proceed, the Admin Personnel will either produce the data requested or engage CTM for fulfillment of the request. Once completed, the Admin Personnel emails the results to the Citizen. As a last step, the Admin Personnel will convert the reported information into a PDF file, and attach the PDF to the request record within the IQ System.

Manage Public Information Requests
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Public Information Office/requestInformation
A Citizen contacts the Public Information Office and requests information from Animal Services.
002 IQ System/createNewRequest
The Public Information Office uses the IQ System to create a new request.
003 Admin Personnel/requestInformation
The Public Information Office contacts the Animal Service Admin Personnel and requests the Citizen-requested information be provided.
004 Provide Resources for People and Pets/generateReport
The Admin Personnel uses the Animal Services system to run reports and analyze/gather information about the feasibility of providing the information requested by the Citizen.
005 Admin Personnel/determineFulfillmentResourceRequirements
The Admin Personnel evaluates the amount of time and effort providing the Citizen-requested information will take, and determines the necessary resource requirements for the data extraction and preparation. The Admin Personnel may consult others within the organization to come up with this estimate.
006 IQ System/addRequestLaborRequirements
The Admin Personnel looks up the request in the IQ system, and annotates the amount of time and effort required to fulfill the Citizen request.
007 Citizen/determineDesiredNextStep
The Admin Personnel contacts the Citizen and requests of the Citizen a "go/no go" determination as to whether Animal Services should proceed with the data extraction, given the resource requirements and the cost associated with the request.
008 CTM/provideDatabaseExtract
The Admin Personnel communicates with CTM and requests that they provision the data requested by the Citizen.
009 Admin Personnel/verifyExtract
The Admin Personnel examines the extracted data provided by CTM, verifies it is what was requested and if it looks appropriate.
010 Admin Personnel/prepareCustomerData
The Admin Personnel uses the system and references whatever means necessary (Excel, Word Documents, hard-copy paper, etc.) to construct the information requested by the Citizen.
011 Admin Personnel/emailResultsToCustomer
The Admin Personnel emails the Citizen the requested information.
012 Admin Personnel/convertResultsToPDF
If possible, the Admin Personnel converts the data to PDF form in preparation for attachment to the IQ System.
013 IQ System/addPDFToPIRRequest
The Admin Personnel uses the IQ System and uploads the PDF version of the data provided to the Citizen to the IQ request record, and marks it as complete.
