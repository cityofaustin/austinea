Update Collection Pay
Use Case Model

Scope: When the collections service takes a payment from the Defendant it records the payment and sends a record to the case management system that includes the payment amount, payment date, payment type. If the collections service determines through their contact with the Defendant that they have a new mail address, email address or phone number then they set a flag indicating that the contact information has changed and include the corrected contact info in the file they send to Muni Court.

This Use case also depends upon the following use cases:

  • Send Message To Collections Use Case

Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Collections/processPayment
The collection service processes a payment on the collections case record. Payments can be made online, by mail and in person if the vendor has a local presence. The payment is referenced by the case number. The payment record includes the payment amount, the payment date and the type of payment (partial or full), starting case balance, case balance, collection fee and is added to the collection receive update object.
002 Collections/addCollectionPaymentUpdate
The collection service processes a payment on the collections case record. The payment is referenced by the case number. The payment record includes the payment amount, the payment date and the type of payment (partial or full), starting case balance, case balance, collection fee and is added to the collection receive update object.
003 Collections/processContactChanges
In the course of corresponding with the defendant by mail, email, text or telephone, if there are any changes in the defendant contact parameters the collection service compiles the changes and includes them in the collection receive changes object.
004 Manage Court Case/Send Message To Collections Use Case
The case management system creates a collections send file (coordinated with the collections vendor) for all cases added that includes the citation information, vehicle information (when applicable), personal identifiers and contact information, case starting balance, case balance, total calculated collection fee and status for any case that is being added or is already in the collections process. The entire data set is transferred to the collection vendor every time there is an update or change to the data. The collection vendor evaluates the entire file and compares the data values for each case to what they have on record. Any changes detected are used to add or update their records.
