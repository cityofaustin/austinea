Restart Collections On Case
Use Case Model

Scope: The case management system checks cases placed into collections that were pauseded because of a motion, appeal or event set on the case subsequent to the case being submitted to the collections system. If all of these pause conditions have processed and/or resolved, the 'days until restart' setting on the case has lapsed and a balance greater than $25 remains on the case then the case management system will send a record to the collections service indicating that the collections can resume.

This Use case also depends upon the following use cases:

  • Send Message To Collections Use Case

001 Manage Court Case/removeCollectionPause
If daysUntilRestart have lapsed and pause conditions are lifted the case management system removes the collection pause flag (sets it to null) in the collection send changes object (referenced by case number). The collection pause flag is set and added to the collection send changes object.
* If the case balance is $0 and the minimum balance is set greater than $0, the case cannot be restarted even if all the other requirements are satisfied.
002 Manage Court Case/Send Message To Collections Use Case
The case management system creates a collections send file (coordinated with the collections vendor) for all cases added that includes the citation information, vehicle information (when applicable), personal identifiers and contact information, case starting balance, case balance, total calculated collection fee and status for any case that is being added or is already in the collections process. The entire data set is transferred to the collection vendor every time there is an update or change to the data. The collection vendor evaluates the entire file and compares the data values for each case to what they have on record. Any changes detected are used to add or update their records.
