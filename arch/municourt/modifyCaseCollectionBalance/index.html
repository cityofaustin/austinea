Modify Case Collection Balance
Use Case Model

Scope: The Judge can review and adjust the fine on a case the has gone to collections. The case management system processes can also add additional fees or costs to the case as it ages: for example: a warrant fee, late payment penalty fee or case processing costs. When these adjustments take place a balance adjustment record is sent to the collection service.

This Use case also depends upon the following use case:

  • Send Message To Collections Use Case

Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Manage Court Case/modifyCaseFine
If the Judge adjusts the fine on the case: The Judge changes the fine amount on the case. The case management system records the new file amount on the case.
002 Manage Court Case/addCostFeeToCase
If the case management system adds fees and costs to the case:
As cost and fees change, the case management can automatically add those to a case.
If the "Time Payment Fee" is not on the case, The case management system adds it and verifies that the case total = the case Judgment amount + time payment fee + additional cost ie: (DPS fee, Collection Fee, Warrant on commitment fee, OMNIBASE administrative fee). (Case Management system processes can add additional fees and costs to a case as the case ages.)
003 Manage Court Case/addModifiedCaseBalance
The case managmenet system adds the case number, starting case balance, case balance (modified case balance) and collection fee to the collection send changes object.
004 Manage Court Case/Send Message To Collections Use Case
The case management system creates a collections send file (coordinated with the collections vendor) for all cases added that includes the citation information, vehicle information (when applicable), personal identifiers and contact information, case starting balance, case balance, total calculated collection fee and status for any case that is being added or is already in the collections process. The entire data set is transferred to the collection vendor every time there is an update or change to the data. The collection vendor evaluates the entire file and compares the data values for each case to what they have on record. Any changes detected are used to add or update their records.
