Change Case Status
Use Case Model

Scope: If the Defendant fails to appear for a court event or fails to comply with conditions set on the case, a "Warrant" (Capias Pro Fine) will be issued. A warrant fee is added to the case. When the case status changes on any non-parking cases to a "Warrant" status or to a status that removes the warrant, a change of status is sent to the collections system. The collection system uses that status to update its communications (by phone) and issued notifications (by mail, email or text) sent to the defendant.

This Use case also depends upon the following use cases:

  • Monitor Case Compliance Use Case
  • Issue Warrants Use Case
  • Send Message To Collections Use Case

Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Monitor Case Compliance Use Case
The Defendant either complies or fails to comply with appearances, payments and sentencing guidelines mandated the law on their case. The case management system monitors the Defendant's compliance by keeping track of appearance dates, payments and the Defendant's timely filing proof of sentence completion.
002 Manage Court Case/evaluateComplianceAction
The case management system evaluates any compliance action generated on the case including:
  • Failure to appear in court as scheduled
  • When the case status changes to and from a warrant status
003 Manage Court Case/Issue Warrants Use Case
When the defendant fails to appear or comply with conditions on the case a warrant is issued.
004 Manage Court Case/addWarrantFee
The case management system adds the warrant fee to the case when the warrant is issued.
005 Manage Court Case/cancelWarrant
When the defendant is brought into compliance with conditions on the case the case management system cancels the warrant on the case. The case status changes from warrant to a status that reflects the previous disposition on the case.
006 Manage Court Case/addCollectionStatus
When case status changes to or from "warrant" the case management system evaluates whether there is an additional fee or changed warrant status on the case and adds the fee and/or changed status to the collection send changes object. New case status: warrant activated or deactivated.
007 Manage Court Case/Send Message To Collections Use Case
The case management system creates a collections send file (coordinated with the collections vendor) for all cases added that includes the citation information, vehicle information (when applicable), personal identifiers and contact information, case starting balance, case balance, total calculated collection fee and status for any case that is being added or is already in the collections process. The entire data set is transferred to the collection vendor every time there is an update or change to the data. The collection vendor evaluates the entire file and compares the data values for each case to what they have on record. Any changes detected are used to add or update their records.
