Modify Collection Fee
Use Case Model

Scope: As required by law, the case management system must recalculate the collection fee as the Defendant receives partial jail time credit or as the Defendant's community service is applied to a case in third party collections. The system must log a statement indicating a collection fee was added, reduced or removed. If reduced, the system must log the reduced amount, date and time. The system must maintain a reference of the original collection fee amount as well.

This Use case also depends upon the following use cases:

  • Verify Jail Credit Use Case
  • Send Message To Collections Use Case

Modify Collection Fee
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Defendant/submitCommunityServiceHours
When receiving credit for community service: The Defendant submits proof (certification) that they have completed working assigned community service hours.
002 Court Clerk/processCommunityServiceHours
The Court Clerk reviews and verifies the submitted community service hours and certification (proof the hours were served).
003 Manage Court Case/applyHoursToCommunityServicePlan
The Clerk selects the Community Plan listed which is associated with the case whose hours are to be credited. The System prompts the Clerk to provide documentation/proof to be scanned. Once the document is scanned, the Clerk verifies the scan is acceptable, and is offered the opportunity to rescan the document should the image not be satisfactory. Once acceptable, the system generates a Receipt which should be stored and printable. The Receipt is locatable and printable by both the case and the community service plan to which the case belongs
004 Court Clerk/presentCommunityServiceHoursReceipt
The receipt for recorded community service hours is sent to a printer that the Court Clerk can access.
005 Defendant/takePaymentReceipt
The Defendant (or whoever made the payment) takes the payment receipt.
006 Manage Court Case/Verify Jail Credit Use Case
When receiving partial jail credit:
The Defendant requests Jail Credit Verification and presents a photo ID to the Court Clerk.
 
If the Defendant has a valid ID:
  • The Court Clerk verifies the person's identity in the case management system and retrieves the PID associated with the Defendant.
  • The Court Clerk runs the jail credit query in Tiburon using the Defendants's name, date of birth, and time frame of incarceration to retrieve the Jail Credit information and confirm that there is Jail Credit.

 
If the Defendant does not have a valid ID to use for confirmation of their identity then the Court Clerk directs the Defendant to the Walk-In Court to determine if the Judge will grant an exception. The Clerk may generate the Tiburon Jail Credit Report for the Defendant using provided demographics and walk it over to the court and present it to the Judge (but not delivering it to the Defendant).
007 Manage Court Case/recalculateCollectionFee
As required by law, the case management system must recalculate the collection fee as partial jail time credit or community service is applied to a case. The case management system recalculates the collection fee.
008 Manage Court Case/logCollectionBalanceModification
The case management system logs a statement indicating a collection fee was added, reduced or removed. If reduced, the system logs the reduced amount, date and time. The system maintains a reference of the original collection fee amount.
009 Manage Court Case/addModifiedCollectionFee
The case management system adds the case number, starting case balance, case balance and recalculated collection fee to the collection send changes object.
010 Manage Court Case/Send Message To Collections Use Case
The case management system creates a collections send file (coordinated with the collections vendor) for all cases added that includes the citation information, vehicle information (when applicable), personal identifiers and contact information, case starting balance, case balance, total calculated collection fee and status for any case that is being added or is already in the collections process. The entire data set is transferred to the collection vendor every time there is an update or change to the data. The collection vendor evaluates the entire file and compares the data values for each case to what they have on record. Any changes detected are used to add or update their records.
