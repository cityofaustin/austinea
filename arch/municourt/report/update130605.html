Table of Interface Descriptions
TypeNameDescription
Web Presence identifyCaseFile Provides secure means of identity for Defendant or Respondent to identify Case File of interest. If there are multiple Case Files, they are presented in a tabular queue for Defendant or Respondent to select Case File of interest.
Manage Court Case retrieveCaseFile Provides interface for the Web Presence to retrieve a Case File Queue (list of available Case Files) based on approved identity criteria from Web Presence.
Web Presence processCaseBalance The Defendant selects to pay the Case File balance and is provided methods to make full payment such as credit card, automated clearing house (ACH) or debit card. Once payment is accepted by the Electronic Financial Service provider in the form of Acknowledgment, the Web Presence processes the case balance with the selected Case File.
Electronic Financial Service processRemittance Electronic Financial Service to process online financial transactions to acknowledge Case File payment. Includes either point of sale (POS) or Web interface to Electronic Financial Service for automated clearing house (ACH), credit or debit card.
Process Criminal Case modifyCriminalCaseFile Used for multiple purposes depending on the disposition of the Case File. Allows the Deputy Court Clerk to make Case File modifications as needed throughout the case review process. Judgment and Receipt are optional outputs. For payments recorded in full, the Deputy Court Clerk distributes payment into cost codes, a plea of no contest is recorded, a waiver of jury trial is recorded, a Receipt is produced, and a Judgment document is generated.
Process Criminal Case addToJudgesQueue Once a Case File is modified to indicate Defendant disposition and payment, the system adds the Case File to the Judges Case File queue for review.
Process Criminal Case generateDPSReport Provides ability to generate Department of Public Safety (DPS) reports on the disposition of criminal cases.
Process Civil Case processCitationInformation Provides interface for automatic vendor updates from transportation and police department. Includes ability for Deputy Court Clerk to manually enter Citation information. The system automatically generates a Case File for each Citation record.
Web Presence scheduleCaseFileDocket The Respondent is provided docket schedule to select scheduled opportunity to dicuss Case File with a Hearing Officer.
Manage Court Case scheduleCaseFile Provides interface to schedule a Case File review with a Hearing Officer or a Judge depending on the disposition of the case. The Case File is automatically added to the appropriate Case File Queue. When the Respondent provides an email address or text message number, the system provides electronic Notification on the time and date to report. Reporting can occur either by showing up in person or via telepresence future capability.
Web Presence initiateTelepresence Provides Respondent ability to initiate a Telepresence session with either a Judge or a Hearing Officer depending on the disposition of the case. The system provides a Wait Time indication or provides indication of the Court's operating hours and provides Respondent an option to schedule the case review at a later time and/or date.
Web Presence connectTelepresence The system automatically intitiaties telepresence session when Case File in queue indicates an awaiting telepresence session.
Manage Court Case reviewCaseFileQueue Provides ability to review either criminal or civil Case File Queue to select ordered Case Files for Case Determination. Case File Queues may include civil or criminal Case Files for review by Hearing Officer, Deputy Court Clerk or Judge.
Manage Court Case processCaseDetermination After review, the Hearing Officer, Deputy Court Clerk, or Judge enters a Case Determination.
Web Presence completeAffidavitForm If Respondent decides to submit an Affidavit demonstrating case situation, the Respondent completes a Web Presence Affidavit form and attaches supporting documentation scanned in PDF format. Relevant information from the Case File automatically populates form fields where applicable.
Manage Court Case updateCaseFile When the Affidavit form is complete to include relevant supporting attachments, the system includes the Affidavit form information and attachments as part of the Case File.
Manage Court Case storeInQue When Case Files require further review, they are stored in a Case File Queue.
Manage Court Case closeCaseFile When remittance is received or disposition is favorable based on Affidavit or other convincing argument, the Case File is closed. Note: under certain accepted dispositions, the Remittance is $0. Future capability includes automatic closure of Case Files when proper remittance is received via Web Presence.
Manage Court Case updateCaseFile Provides updated information to the Case File on Appeal information from Respondent.
