Create User Account
Use Case

Use Case Scope

The Defendant accesses the web presence and uses their Citation to locate their case. The system determines the available options associated with the case. The Defendant chooses an option requiring them to log onto the system. Logins are in the form of last name and a security code. If the Defendant does not have a security code already registered with their PID record, the system prompts the Defendant which method they prefer to use for notification; Voice messaging or Text message. Text messaging will only be offered if a mobile phone number is on the current PID record, otherwise the system will use voice messaging, and not offer the Defendant a choice of verification method. The System notifies the Defendant of the last 4 digits of the phone number associated with the citation being used for the verification. If the phone of record is a mobile phone, and text messaging has been specified as the preferential verification method, a text message is sent to the mobile phone. If the phone is a home number, an automated voice message will be used to send a recorded voice with the security code information. This security code is valid for 24 hours (the amount of time the security code is valid is configurable.) Once the Verification has been sent, the system associates the security code with the PID.

Type/InterfaceDescription
Web Presence/
identifyCaseFile		 Provides a legal means of identity for Defendant or Respondent to identify Case File of interest. If there are multiple Case Files, they are presented in a tabular queue for Defendant or Respondent to select Case File of interest.
Manage Court Case/
retrieveCaseFileByCaseNumber		 The Web Presence interfaces with the system to retrieve a Case File based on a case number provided.
Manage Court Case/
determineOption		 When a user selects a particular Case File of interest from the Web Presence, the system determines the various options available to the User. The system provides a Municipal Court staff configurable rules engine to allow Court Staff to modify business rules associated with the options provided to the User. The rules engine provides a customizable response based on current Case File attributes such as warrant status, age of Defendant or Respondent, attorney assigned to the case, and plea state (just to name a few) leading to the options provided to the User. Depending on Municipal Court Staff needs, some responses are non-selectable and grayed out providing indication to the User why the legal option is not selectable. Because laws frequently change, it's important the options are not hard coded into the case management system; rather, the system allows the Court Staff to make appropriate option changes using customizable rules filter.
Web Presence/
selectOption		 Based on a customizable Option rules engine, the User is provided a list of options to proceed with the Case File. To reduce confusion, some of the options (determined by Court Staff) are grayed out and non-selectable with an explanation as to why the Option is not available. When selecting the option, the user is provided a confirmation of the legal option selected.
Manage Court Case/
setOption		 The User is offered a set of options and makes a selection. The Case management system is notified of the option selected.
Web Presence/
provideUsernameAndPassword		 A request or condition has occurred which requires identification of the User. The Web Presence prompts the user for their Last name and their Security Code. If the defendant does not have a Security code, they are offered the option to establish one.
Web Presence/
provideVerificationPreference		 The system displays a message indicating a security code will be sent within "x number of minutes" (configurable) to the user. The system additionally lists the last 4 digits of the phone number associated with the case, to which the message will be sent. If the contact number is a mobile phone, the system requests a preference for code delivery. The user selects text message or voice system communication (IVR). If the contact number is a not a mobile phone, the system sets the verification preference to voice system communication by default.
Manage Court Case/
sendSecurityCode		 Based on the preference of verification, the system generates a random security code and sends it to the phone number of record for the case using the verification preference specified by the user. The user can specify either a text message or automated voice communicated message (IVR) as their preference.
Manage Court Case/
updateSecurityCodeOnPID		 A new security code has been created for a user. The system updates the PID record of the associated defendant, with the new Security code specified.
