Muni Court Team,

After losing some sleep over Matt's challenge to figure out a way to queue remittance and allow for Defendants to process multiple Case Files, I cracked the code. The problem wasn't as challenging as anticipated. To follow my discussion, please select the Set Criminal Case on Docket Interaction Overview (activity model). The Interaction Overview shows how all the sequence diagrams interact or tie together. Using Aaron Brown's business Lego discussion, the Interaction Overview provides a way to show how all the processes fit together. Each activity (blue, round corner rectangles) on the diagram equals a sequence diagram, which can be viewed as a specific activity. Each of the sequence diagrams are depicted later in this report to include their interface descriptions in the table below the diagram.

The Interaction Overview identifies the business rules leading to the sequence of events shown on the sequence diagram. Also note, there are business rules captured on the sequence diagram (i.e., opt, alt, etc.) demonstrating how complex some processes can get - i.e., business rules inside more business rules. This is real life and the power of the Unified Modeling Language (UML) that traditional modeling approaches are unable to depict.

On further inspection, it would appear that this use case is more than Set Criminal Case on Docket, because only one leg of the flow has anything to do with the Docket (i.e., Establish Not Guilty Plea). Therefore, I'd suggest the use case should be renamed to Establish Case File Legal Option or something like that. We can discuss during our next meeting.

Use Case Specification
Set Criminal Case on Docket

Select Legal Option Scope

The Defendant enters relevant legally binding Identity information into the system to access a list of case files and selects the Case File of interest (note: the Case File list could be just one item). Based on the type of Case File, the system provides the Defendant various Legal Options and confirms the Defendant selection.

Web Presence/
identifyCaseFile		 Provides a legal means of identity for Defendant or Respondent to identify Case File of interest. If there are multiple Case Files, they are presented in a tabular queue for Defendant or Respondent to select Case File of interest.

retrieveCaseFile		 The Web Presence interfaces with the system to legally validate Defendant Identity and retrieve selected Case File from a Case File Queue.
Manage Court Case/
determineLegalOptions		 When a Defendant or Respondent selects a particular Case File of interest from the Web Presence Case File Queue, the system determines the various legal options available to the Defendant or Respondent. The rules engine provides a customizable response based on current Case File attributes such as warrant status, age of Defendant or Respondent, attorney assigned to the case, and plea state (just to name a few) leading to the legal options provided to the Defendant or Respondent. Depending on Municipal Court Staff needs, some responses will be non-selectable and grayed out providing indication to Defendant or Respondent why the legal option is not selectable. Because laws frequently change, it's important that the legal options are not hard coded into the case management system; rather, the system should allow the Court Staff to make appropriate legal option changes using customizable rules filter.

selectLegalOption		 Based on a customizable Legal Option rules engine, the Defendant or Respondent is provided a list of legal options to proceed with the Case File. To reduce confusion, some of the legal options (determined by Court Staff) will be grayed out and non-selectable with an explanation as to why the Legal Option is not available. When selecting the legal option, the user will need to provide confirming the legal option selected.
Web Presence/
setLegalOption		 The system sets the Legal Option selected with the associated Case File.
Manage Court Case/
confirmLegalSelection		 The Defendant confirms a Legal Selection in the form of an Acknowledgment for selected Case File (under some circumstances the Plea state is null such as when selecting compliance dismissal request) and the system provides a confirmation Acknowledgment. The Defendant may decline the Acknowledgment and return to Select Legal Option. The Acknowledgment statement is based on the Legal Option selected and is modifiable by the Municipal Court staff.
Manage Court Case/
setLegalOptionState		 The Web Presence returns an Acknowledgment and if accepted by the Defendant, the system sets the Legal Option state in the system.

Establish Not Guilty Plea Scope

When a not guilty plea is selected as a Legal Option, the Defendant is presented with types of Dockets to select from. When a particular Docket is selected, the system provides a Docket Schedule for the Defendant to select as a preference. An Acknowledgment of Jury Waiver is confirmed and the Docket Schedule selected and Case File is added to the Complaint Queue for future review by Court staff. The system also provides an option to request additional assistance for Defendants requiring special needs access or language interpretation.

Establish Deferral Request Scope

Establish Compliance Dismissal Request Scope

When the Defendant requests a Compliance Dismissal, the system provides a list of Method of Compliance for selection. When Compliance Documentation is required, the system provides a means to upload Compliance Documentation. When additional compliance information is required, the system provides a form to gather the additional information from the Defendant. A Compliance Confirmation confirms legal commitment by the Defendant and the system stores the Compliance Documents and/or Compliance Information with the associated Case File in the Compliance Documentation Queue for future review by the Court staff.

Complete Case File Activity Scope

On completion of selected Case File Legal Options, the Defendant is presented with the option to review and select another Case File from the Case File Queue, process Case File Remittance balance (if any), or stop the current session.
