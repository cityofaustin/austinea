Accept Case File For Document
Use Case

Use Case Scope

Using the Web Presence, the Document Submitter selects the option to submit documentation. The Document submitter selects type of document from a predefined list derived by the business rule engine, which is then stored with the case file along with the document in question. The document is stored in the Document Submission queue and is reviewed by the Court Clerk. The document must be an image file, video recording, audio recording, Office Documents, etc, using industry standard formats. The Document Submitter is notified of document receipt, and the document is verified by the Court Clerk. If all outstanding elements of the case are provided, the Court Clerk places the Case in the Judge's queue. For cases forwarded to the Judge, the Judge makes one of two decisions, either to reject the case or to dismiss the case. If the case is rejected, the Judge adds comments to the case file and sends it back to the Court clerk. If the Case is dismissed, the case file is modified to reflect that the case is now terminated. In both cases, the Document Submitter and/or the appropriate contactee are sent notification of the state of the case following the Judge's determination.

Prior to the message on the sequence diagram is a Reference item [Ref] essentially identifying a preconditional element to the sequence events where the Motion Filer established the File Motion option. A summary of the reference follows:

Ref: 1. Select Legal Option

The system provides the ability for a Respondent or Defendant to manage all Case Files identified in the case management system using a Web Presence online. The system provides a legally binding form of identification to be persistent throughout the Web session. When a Defendant or Respondent selects a particular Case File of interest from the Web Presence Case File Queue, the system determines the various legal options available to the Defendant or Respondent. The system provides a Municipal Court staff a configurable rules engine to allow Court Staff to modify business rules associated with the options provided to the Defendant or Respondent via Web Presence. The rules engine provides a customizable response based on current Case File attributes such as warrant status, age of Defendant or Respondent, attorney assigned to the case, and plea state (just to name a few) leading to the legal options provided to the Defendant or Respondent. Depending on Municipal Court Staff needs, some responses are non-selectable and grayed out providing indication to Defendant or Respondent why the legal option is not selectable. Because laws frequently change, it's important the legal options are not hard coded into the case management system; rather, the system allows the Court Staff to make appropriate legal option changes using customizable rules filters. Once a Legal Option is selected, the system confirms the Legal Option and it's implication to Respondent or Defendant before storing it with the Case File.

Accept Case File Document Interface Descriptions

Type/InterfaceDescription
Web Presence/
selectDocumentType		 The online user is offered the appropriate types of Documents they can file legally based on the details of their case and dictated by the rules engine.
Web Presence/
provideDocument		 The Web Presence prompts the Document Submitter to specify the document they would like to append to their case, and uploads the document.
Manage Court Case/
attachDocumentToCase		 The system takes the document which has been uploaded and attaches it to the case file.
Manage Court Case/
storeInClerkQueue		 The system adds a changed Case in a queue for a Court Clerk.
Manage Court Case/
sendNotificationToDocumentSubmitter		 Based on the identity logged into the web presence, the system will send a notification to the Document Submitted indicating the case, the document attached, and any other relevant information appropriate as necessary for the court. This notification is an email, text message, automated phone message, or any other notification mechanism as dictated by the business rule engine. This is used for notifications about attached documents, notifications about Case Dismissal, or notifications about changes in status of the case as deemed appropriate by the business rule engine.
Manage Court Case/
selectCaseFromCourtClerkQueue		 The Court Clerk identifies and selects a Case from their pending queue. They see all associated forms, attachments, and signatures associate with the case selected.
Deputy Court Clerk/
evaluateCaseDocuments		 The Court Clerk evaluates the Case and all it's associated attachments, signatures, elements and determines if the case is ready to be forwarded to the Judge on the Case.
Manage Court Case/
signAndStoreInJudgeQueue		 When County Clerk has verified the document, attachments, signatures, etc. of the court case and has determined it is prepared to be sent to the Judge, the system attaches the Court Clerk's signature to the Case File and places the Case in the Judge's queue.
Manage Court Case/
selectCaseFromJudgeQueue		 The Judge identifies and selects a Case from their pending queue. They see all associated forms, attachments, and signatures associate with the case selected.
Manage Court Case/
dismissOrRejectCase		 The Judge reviews all elements of the case file, and selects one of two Case Rulings: "Dismiss" or "Reject" on the case selected from their work queue. The system attaches their signature,and adds the Judge's Ruling to the case. If the Judge rejects the case, they will annotate the case file with further instructions for the County Clerk. Dismissing the case indicates the case file is now completed, and the matter is closed.
