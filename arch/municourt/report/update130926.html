Monitor Performance
Use Case

Use Case Scope

System admin logs in, accesses performance measurements dashboard and specifies a date range for which to examine system health and performance.  The system admin can export the information to a CSV Excel spreadsheet for further analysis.  The system admin can configure various performance metrics and alerts (pager, email, text) to be tracked and can specify acceptable performance limits that are associated with relevant indicators.  In addition, the performance monitoring will provide:

    CPU % usage of all users logged into the system as well as the system server software itself.
    Read/write Disk status
    Logged Failures
    Timing for specific workflow jobs

displayPerformanceInformation		 The system uses the start date/time information and the end date/time information to display metrics related to the performance of the system. The system provides a dashboard of activities currently being monitored by the system. The system administrator can configure and modify which metrics are displayed (a list provides available performance metrics.) Each metric has a configurable acceptable ranges for metrics, and dashboard alerting. The system administrator can designate when performance falls outside of the configured ranges for on screen color (green for okay, red for outside.) Additionally, the system allows the administrator to designate a phone number for text alerting, and messages the administrator when metrics are outside the acceptable range specified. The list of metrics available for monitoring should include but not be limited to:
    CPU % Usage for each logged on user
    Time for Reads / Writes to each Table
    Disk Failure reporting
    Specific timing for any work flow element (...I.E. how long to post to a judges queue)

The system performance monitoring also allows for the export of any information displayed to EXCEL CSV format.
