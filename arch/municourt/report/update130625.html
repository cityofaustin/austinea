06/12/13 - Select Legal Option

Previously, we discussed how a Web Presence could provide the ability for a Defendant to select their legal option. Last week we pursued the legal option of "Not Guilty". The following sequence diagram is the triggering process we created a couple of weeks ago to Process Not Guilty Plea discussed below. During our next meeting, we'll examine other Defendant Plea processes triggered by Select Web Presence Legal Option.


The following sequence diagram depicts the Web interface of the Not Guilty Plea and the table below provides the interface descriptions.

Note: Select image for larger view.

Process Not Guilty Plea

Precondition: The Defendant identifies the selected Case File Plea of not guilty from Select Web Presence Legal Option.

Table of Interface Descriptions

Type/InterfaceDescription
Web Presence/
confirmPlea		 The Defendant confirms a Plea state of not guilty for selected Case File and the system provides a confirmation Acknowledgment of the not guilty Plea. The selected Case File is updated to include the not guilty Plea state.
Manage Court Case/
setPleaState		 The Web Presence interfaces with the system and sets the not guilty Plea state on the Case File and returns an Acknowledgment allowing the Defendant an opportunity to change their Plea. If a Plea change is selected, the Defendant is returned to the Legal Options view.
Web Presence/
selectDocketType		 The system provides the Defendant the option to select the Docket type (Appearance, Bench Trial or Jury Trial) with full explanation of each Docket type. When a Docket Type is selected, the system returns the Docket Schedule based on the Docket type selected.
Manage Court Case/
identifyDocketSchedule		 Based on the Docket type selected, the system returns the date, time, and court room information regarding Docket time(s) available. The system provides the Court Staff selectable method to return available dockets from which to choose - such as next available, one or more available, all available. The Defendant selects the specific Docket based on the system offerings.
Web Presence/
selectDocketSchedule		 The Defendant selects the desired Docket Schedule item from the schedule offerings.
Manage Court Case/
setDocketSchedule		 Interface from Web Presence to system that establishes the selected Docket Schedule.
Manage Court Case/
addToComplaintQueue		 The system adds the Docket Schedule Case File to the Complaint Queue.
Web Presence/
acknowledgeJuryWaiver		 If Defendant selects bench trial, the system requires an Acknowledgement of a Jury Waiver stored in the Case File.
Web Presence/
requestAdditionalRequirements		 When the Defendant requires special needs (i.e., interpreter, handicap access, etc.), the system provides special needs options for selection by the Defendant.
Manage Court Case/
addToAdditionalRequirementsQueue		 Interface to store special needs request into the system Case File.
