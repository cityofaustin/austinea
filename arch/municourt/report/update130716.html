Criticality Level Definitions

All requirements are assigned one of the following three criticality levels:

  • Must Have - these requirements are considered highly critical and must be satisfied by the CMS upgrade in some way.
  • Expected - these requirements are somewhat critical to the end users of the CMS upgrade.
  • Desired - these requirements would add value but are not critical.
