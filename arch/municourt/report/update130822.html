File For Representation
Use Case

Use Case Scope

The Defense Attorney uses the web presence to locate their case. The web presence provides case information. The Attorney chooses an option for 'Filing Representation' on a specific case (if it is offered.) The web presence provides two options "new Attorney" and "sign in". If The Attorney is already vetted and has a password, they use the "Sign in" to log onto the system. If the Attorney is already qualified on the system, and there is no representation on the case already, and there is no disposition on the case, the Attorney is assigned as a case party. The case file is updated with the information that web activity has occurred, and an email is sent to the Attorney confirming the change to the case.

For a new Attorney who has not be previously vetted, and has no current password for the system, the Attorney is presented a form to fill out. The form is pre-populated with Case information. The Attorney provides their name and Bar Card Number. Once the form is submitted, the Attorney's application is added to a "New Attorney Queue". An Email is generated to the Attorney with a confirmation of submission. The Attorney will be called/vetted, and a password is set for their future use on the system.

File For Representation Interface Descriptions

Type/InterfaceDescription
Web Presence/
requestCaseFile		 The system prompts for a case file number. No identity is required for retrieval of case information, as all details displayed are public record.
Manage Court Case/
retrieveCaseFileByCaseNumber		 The Web Presence interfaces with the system to retrieve a Case File based on a case number provided.
Manage Court Case/
determineOption		 When a user selects a particular Case File of interest from the Web Presence, the system determines the various options available to the User. The system provides a Municipal Court staff configurable rules engine to allow Court Staff to modify business rules associated with the options provided to the User. The rules engine provides a customizable response based on current Case File attributes such as warrant status, age of Defendant or Respondent, attorney assigned to the case, and plea state (just to name a few) leading to the options provided to the User. Depending on Municipal Court Staff needs, some responses will be non-selectable and grayed out providing indication to the User why the legal option is not selectable. Because laws frequently change, it's important the options are not hard coded into the case management system; rather, the system should allow the Court Staff to make appropriate option changes using customizable rules filter.
Web Presence/
selectOption		 Based on a customizable Option rules engine, the User is provided a list of options to proceed with the Case File. To reduce confusion, some of the options (determined by Court Staff) will be grayed out and non-selectable with an explanation as to why the Option is not available. When selecting the option, the user will need to provide confirmation of the legal option selected.
Manage Court Case/
fileForRepresentation		 The Web Presence has provided information about a case and indicates a user has specified they would like to file for Representation for the case. This option will only be offered if the configurable rules engine dictates a ruleset in which the option should appear. The option will not be selectable if there is already representation for the case, or if there is no disposition on the case.
Web Presence/
identifyAttorney		 The system presents a login to an Attorney for validation of their identity. It presents two options; either a logon id / Password, or the option to establish themselves as a new Attorney on the system.
Manage Court Case/
updateCaseRepresentation		 The system assigns a validated Attorney to a case file. In order for the system to successfully assign the Attorney, the Attorney must already be qualified on the system, there is no representation on the case already, and there is no disposition on the case at the time of assignment.
Manage Court Case/
provideNewAttorneyForm		 The Web Presence receives a request to establish a new Attorney on the system, and requests the Court Management System provide the mechanism by which Attorneys register.
Web Presence/
completeNewAttorneyForm		 The system presents a form of fields for an applicant to complete related to establishing themselves as an Attorney within the system. The Attorney completes the form (application) and on completion and submission, the system presents a completed view for final acceptance into the system./td>
Manage Court Case/
AddAttorneyToNewAttorneyQueue		 A new Attorney application has been completed and is placed in the New Attorney Queue for review and validation. Note: There is a desire for this functionality to become automated in the future, with an interface to the Texas State Bar Association.
Manage Court Case/
sendNotificationToAttorney		 The System generates a notification in response to either a New Attorney Application or a Request to File for Representation on a case. The system sends notification to the address linked (and verified) with the Attorney's identity. The system records the date and time the notification is sent to the Attorney. If an Attorney has filed for representation on a specific case, the Email provides a summary of case information, including but not limited to the Attorney of Representation. If the notification is in response to a New Attorney Application, the date and time the application was submitted to the new Attorney queue is provided.
