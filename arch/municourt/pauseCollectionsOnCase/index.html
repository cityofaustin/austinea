Pause Collections On Case
Use Case Model

Scope: The case management system monitors the case sent to collections for any 'pause collections' condition. If a Defendant files a motion or appeal on a case, or if an event (appearance for example) is scheduled after the case has been sent to collections then the case management system sends a 'pause collections' message to the collections service.

This Use case also depends upon the following use cases:

Pause Collections On Case
Sequence Diagram

Req#Type/Interface - Requirement DescriptionPerformance
001 Court Clerk/processMotionOrAppealOnCase
The Defendant (or representation) files a motion or appeal on the case in the form of a document or form presented to the Court Clerk that references the case by the case number. The Court Clerk processes the motion or appeal document.
002 Manage Court Case/scanMotionToCase
Whan a motion is being filed: The Clerk uses the case management system to scan the motion onto the case by running the document through a scanner which creates an image of the document and a corresponding record with the case number, the motion type, the date the image was created and a link to where the image is stored on the SAN.
003 Manage Court Case/enterAppealAction
When an appeal is being filed: The Clerk uses the case management system to enter an appeal action on the case.
004 Manage Court Case/attachAppealDocumentation
The Clerk scans the appeal document onto the case. The case management system attaches the document to the case by creating a record that includes the case number, the document type, the date the image was created and a link to the location of the image on the SAN.
005 Manage Court Case/setCaseOnDocket
When Judge sets an appearance: The Judge orders an appearance to review the case or the Court Clerk schedules a court event. The case management system sets the date on the docket calendar when the Defendant must appear in court before the Judge.
006 Manage Court Case/setCaseOnDocket
When Clerk sets event on case: The Judge orders an appearance to review the case or the Court Clerk schedules a court event. The case management system sets the date on the docket calendar when the Defendant must appear in court before the Judge.
007 Manage Court Case/addCollectionPause
If action on case is a motion, appeal or scheduled event: The case management system sets the value of the collection pause flag to 'true' in the collection send changes object (referencing the case to be paused by case number). Pausing collections means temporarily ceasing collection activity. If the case is paused for an appeal the vendor may not collect or accept payment until the appeal pause is lifted. Under other pause conditions the vendor ceases collection activity but may accept payment.
 
Pause Codes:
  • APPEAL and APPEALRL (RL is for Red light cases) = Pause Appeal
008 Manage Court Case/Send Message To Collections Use Case
The case management system creates a collections send file (coordinated with the collections vendor) for all cases added that includes the citation information, vehicle information (when applicable), personal identifiers and contact information, case starting balance, case balance, total calculated collection fee and status for any case that is being added or is already in the collections process. The entire data set is transferred to the collection vendor every time there is an update or change to the data. The collection vendor evaluates the entire file and compares the data values for each case to what they have on record. Any changes detected are used to add or update their records.
